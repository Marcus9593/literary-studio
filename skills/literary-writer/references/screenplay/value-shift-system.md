# 价值转变系统（Value Shift System）

> 基于罗伯特·麦基《故事：材质、结构、风格和银幕剧作的原理》的价值转变理论。
> 本文件定义故事中价值维度的识别、标注与分析方法，用于指导叙事张力的构建与检测。

---

## 二、价值转变标注方法

### 2.1 转变类型（Shift Type）

| 类型 | 标识 | 定义 | 示例 |
|------|------|------|------|
| 正向转变 | `positive` | 从负极端向正极端移动 | 从"绝望"到"希望" |
| 负向转变 | `negative` | 从正极端向负极端移动 | 从"安全"到"危险" |
| 讽刺转变 | `ironic` | 表面正向、实则负向（或反之） | 获得"自由"却陷入更深的"孤独" |
| 横向转变 | `lateral` | 同一价值等级内的维度切换 | 从"忠诚"到"正义"（价值冲突） |

### 2.2 标注格式

每个价值转变事件应标注以下字段：

```
dimension:    价值维度对（如 "生/死"）
from_value:   转变前的状态值（如 "安全"）
to_value:     转变后的状态值（如 "危险"）
shift_type:   positive | negative | ironic | lateral
magnitude:    微变 | 显著 | 颠覆
subject:      承受转变的角色或实体
chapter:      转变发生的章节
scene_ref:    关联场景描述
```

### 2.3 讽刺转变的特殊处理

讽刺转变（Ironic Shift）是麦基理论中最有力的叙事工具之一：

- **正向讽刺**：角色获得了他想要的，但发现这并非他需要的
  - 例：主角赢得比赛（成功），却发现胜利建立在背叛朋友之上（失去归属）
- **负向讽刺**：角色失去了什么，却意外获得了更深层的满足
  - 例：主角破产（失败），却找回了与家人的连接（归属）
- **双重讽刺**：两个角色的价值转变互为镜像
  - 例：A获得了B失去的，B获得了A失去的

## 三、转变幅度评估

### 3.1 幅度等级

| 等级 | 标识 | 量化标准 | 叙事功能 |
|------|------|----------|----------|
| 微变 | `micro` | 状态在同侧小幅波动 | 维持张力、铺垫情绪 |
| 显著 | `significant` | 跨越价值中线（从正到负或反之） | 推动剧情、改变格局 |
| 颠覆 | `颠覆` | 价值维度本身的重新定义 | 高潮反转、主题升华 |

### 3.2 幅度判断指南

**微变**（Micro）：
- 角色在同一价值极内的状态波动
- 例：从"稍微安全"到"更安全"，或从"有点焦虑"到"更加焦虑"
- 功能：积累情绪张力，为大转变做铺垫

**显著**（Significant）：
- 角色跨越了价值正负的中线
- 例：从"安全"到"危险"，从"被爱"到"被背叛"
- 功能：构成场景的核心转折，推动剧情进入新阶段

**颠覆**（Reversal / 颠覆性转变）：
- 价值维度本身被重新定义，或多个维度同时发生剧烈转变
- 例：角色发现他一直追求的"真相"本身就是谎言（真相/谎言维度的元层翻转）
- 功能：构成故事高潮，实现主题升华

## 四、价值曲线与叙事张力

### 4.1 价值曲线定义

价值曲线（Value Curve）是将故事中逐场景/逐章的价值转变幅度绘制为时间序列的可视化工具。

```
幅度
 ↑
颠覆 │                              ★ 高潮
     │                         ╱╲
显著 │        ╱╲              ╱    ╲
     │   ╱╲  ╱  ╲    ╱╲     ╱      ╲
微变 │  ╱  ╲╱    ╲  ╱  ╲   ╱        ╲
     │ ╱          ╲╱    ╲ ╱          ╲
  0  │──────────────────────────────────→ 章节
     │
负向 │
```

### 4.2 张力曲线与价值曲线的关系

| 张力模式 | 价值曲线特征 | 适用场景 |
|----------|-------------|----------|
| 渐升式 | 逐章微变累积，最终显著/颠覆 | 经典三幕结构 |
| 波浪式 | 正负交替，幅度递增 | 连续剧、长篇连载 |
| 阶梯式 | 平台期+突变，每阶更高 | 升级流、竞技 |
| 断崖式 | 持续正向后突然颠覆 | 悲剧、反套路 |
| 反弹式 | 持续负向后绝地逆转 | 爽文、励志 |

### 4.3 理想价值曲线（麦基模型）

```
第一幕                第二幕                     第三幕
┌─────────────┐  ┌──────────────────────────┐  ┌──────────────┐
│ 建立初始状态  │  │ 升级冲突 + 中点反转       │  │ 最终对决     │
│              │  │                          │  │              │
│ 微变 ×2-3   │  │ 微变 → 显著 → 微变       │  │ 显著 → 颠覆  │
│ 播种价值维度  │  │ 中点：显著转变（假胜利）  │  │ 高潮：颠覆   │
│              │  │ 至暗：显著负向转变        │  │ 结局：最终   │
└─────────────┘  └──────────────────────────┘  └──────────────┘
```

### 4.4 章节级价值密度

- **高密度章节**（每章 2+ 个价值转变）：高潮段、冲突集中段
- **中密度章节**（每章 1 个价值转变）：正常推进
- **低密度章节**（0 个价值转变）：需警惕"平坦区"，可能失去读者

## 五、与现有系统的集成

### 5.1 与故事事件系统集成

价值转变通过 `value_shift_occurred` 事件类型纳入故事事件体系：

```python
# story_event_schema.py
class StoryEvent(BaseModel):
    event_type: Literal[
        ...,
        "value_shift_occurred",  # 新增
    ]
```

### 5.2 与弧光分析集成

价值转变是角色弧光的微观表达：

| 弧光阶段 | 典型价值转变 |
|----------|-------------|
| 起点 | 建立初始价值状态（正或负） |
| 依赖期 | 微变，角色在旧模式中挣扎 |
| 抉择期 | 显著转变，角色被迫面对核心矛盾 |
| 觉醒期 | 显著正向或讽刺转变 |
| 蜕变期 | 颠覆性转变，完成弧光 |

### 5.3 与冲突分析集成

冲突分析检测"什么力量在对抗"，价值转变回答"对抗导致了什么变化"：

- 每个冲突事件应关联至少一个价值转变
- 高冲突密度 + 低价值转变 = "假冲突"（有对抗但无实质进展）
- 价值转变的幅度应与冲突强度正相关

### 5.4 与伏笔系统集成

伏笔的埋设与回收可视为跨时间的价值转变：

- 伏笔埋设：播种一个未来的价值转变（如"真相/谎言"维度的微变暗示）
- 伏笔回收：兑现为显著或颠覆性转变

### 5.5 与行动建议系统集成

当系统检测到以下问题时，应生成价值转变相关的行动建议：

| 检测结果 | 行动建议 |
|----------|----------|
| 连续 3+ 章无价值转变 | 建议增加一个显著转变事件 |
| 价值曲线过于平坦 | 建议在中段加入颠覆性反转 |
| 只有正向转变 | 建议加入负向转变以增加张力 |
| 讽刺转变为零 | 建议利用讽刺转变深化主题 |

## 六、自检清单

### 6.1 场景级自检

- [ ] 本场景涉及哪个价值维度？
- [ ] 场景结束时，该维度的状态是否与场景开始时不同？
- [ ] 转变类型是什么？（正向/负向/讽刺/横向）
- [ ] 转变幅度是否与场景的叙事重要性匹配？
- [ ] 如果没有价值转变，这个场景存在的理由是什么？

### 6.2 章节级自检

- [ ] 本章是否至少包含一个价值转变事件？
- [ ] 本章的价值转变是否推进了主线或重要副线？
- [ ] 本章的价值转变幅度是否与前一章形成有意义的对比？
- [ ] 是否存在连续多章的价值曲线平坦区？

### 6.3 故事级自检

- [ ] 故事是否使用了至少 3 个不同的价值维度？
- [ ] 主角经历的价值转变是否构成完整的弧光？
- [ ] 高潮处是否有至少一个颠覆性转变？
- [ ] 结局的价值状态是否与开场形成有意义的对照？
- [ ] 价值曲线的整体形状是否与故事类型匹配？
- [ ] 是否有讽刺转变来深化主题？
- [ ] 副线的价值转变是否与主线有主题呼应？

### 6.4 系统级自检（自动化）

- [ ] 每个 `value_shift_occurred` 事件是否包含完整的 dimension/from/to/type/magnitude 字段？
- [ ] 价值曲线数据是否与章节事件日志一致？
- [ ] 检测到的"平坦区"是否已生成行动建议？
- [ ] 价值分析结果是否已写入 `understanding/value_shifts.json`？
