# 图书/非虚构创作指南

## 核心原则

非虚构作品的目标是：**用清晰的结构传递有价值的信息，让读者有所收获。**

三种核心价值：
- **知识传递**：教会读者什么
- **认知升级**：改变读者的思维方式
- **行动指引**：帮助读者做出决策或采取行动

## 图书类型

### 商业/管理类
- 结构：问题→分析→框架→案例→行动建议
- 特点：数据支撑、案例丰富、可操作性强
- 示例：《从0到1》《原则》《创新者的窘境》

### 自我成长类
- 结构：痛点→原理→方法→练习→案例
- 特点：科学依据、个人故事、实用工具
- 示例：《原子习惯》《心流》《刻意练习》

### 科普类
- 结构：引人入胜的开头→逐步深入→核心洞见→启示
- 特点：类比生动、故事驱动、深入浅出
- 示例：《人类简史》《时间简史》《思考，快与慢》

### 传记/回忆录
- 结构：时间线 + 主题线交织
- 特点：细节丰富、情感真挚、有时代感
- 特殊要求：事实准确、引用有据

### 技术/专业类
- 结构：由浅入深、循序渐进
- 特点：逻辑严密、图表辅助、练习配套
- 特殊要求：术语准确、代码/公式正确

## 结构设计

### 金字塔结构
```
核心主题（一句话概括全书）
├── 第一部分：基础/背景
│   ├── 第1章：为什么重要
│   ├── 第2章：基本概念
│   └── 第3章：历史脉络
├── 第二部分：核心内容
│   ├── 第4章：原理/框架
│   ├── 第5章：方法论
│   └── 第6章：案例分析
├── 第三部分：实践应用
│   ├── 第7章：如何开始
│   ├── 第8章：常见问题
│   └── 第9章：进阶路径
└── 结语：总结与展望
```

### 问题导向结构
1. 提出核心问题
2. 展示现有答案的不足
3. 提出新框架/新视角
4. 用案例和数据支撑
5. 给出行动建议

## 章节写作

### 章节模板
```
章节标题：[动词+名词，如"建立你的第一原则"]

开头钩子（300-500字）
- 一个引人入胜的故事/案例/问题
- 让读者明白"这章能给我什么"

核心内容（2000-4000字）
- 概念解释（用类比和例子）
- 框架/模型展示
- 数据/研究支撑
- 实际案例

行动建议（500-1000字）
- 具体可执行的步骤
- 常见错误提醒
- 练习/自查清单

章节总结（200-300字）
- 核心要点回顾
- 与下一章的衔接
```

## 写作技巧

### 深入浅出
- **类比**：用读者熟悉的事物解释陌生概念
- **故事**：用具体案例承载抽象理论
- **可视化**：图表、流程图、对比表
- **渐进式**：先给全景，再深入细节

### 可信度建设
- **数据引用**：标明来源，使用最新数据
- **专家观点**：引用权威人士的见解
- **案例真实性**：注明案例来源和时间
- **承认局限**：不夸大结论，承认不确定性

### 读者体验
- **每章独立可读**：读者可能跳着读
- **过渡衔接**：章与章之间有逻辑连接
- **重复关键信息**：重要概念在不同章节从不同角度重申
- **留白**：给读者思考的空间

## 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 读起来像论文 | 学术腔太重 | 用对话式语言，多讲故事 |
| 信息过载 | 贪多求全 | 聚焦核心论点，删除次要内容 |
| 结论空洞 | 缺乏具体建议 | 给出可执行的行动步骤 |
| 案例陈旧 | 使用过时素材 | 更新案例，使用近期数据 |
| 结构混乱 | 逻辑不清 | 每章只讲一个核心点 |
