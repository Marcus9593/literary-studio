## 一、剧本格式规范

### 1.1 中国影视行业标准格式

#### 场景标题
```
1. 咖啡馆内 - 日
2. 医院走廊 - 夜
3. 城市街道 - 黄昏（雨）
```
- 格式：`序号. 地点 + 内/外 + 时间（天气/特殊条件）`
- 序号用于拍摄场次单（不按剧情顺序，按拍摄顺序编号时另起场次表）
- 地点要具体：不是"房间"，是"老式筒子楼 - 李明的出租屋"

#### 动作线（描写段）
- 现在时态，主动语态
- 简洁、可视化——只写摄影机能拍到的
- 不写内心独白（除非有画外音/旁白设定）
- 不写"我们看到""镜头转向"等导演指令（除非你是导演兼编剧）
- 每段不超过 4 行

#### 对白格式
```
李明
（压低声音，手指敲着桌面）
我知道你在撒谎。
```
- 角色名居中加粗或大写
- 括号内为表演提示（尽量少用，好演员不需要你教怎么演）
- 对白正文

#### 对白精简规则
- **剧本对白比小说对白短 30%**——同一句话在剧本中必须更精简，删掉修饰语和解释性词语
- **沉默和动作承担叙事**——角色不说话时，用动作/表情/沉默传递信息（"她没回答，把杯子推到桌边"）
- **一句对白只传递一层信息**——不要在一句台词里塞两个以上的信息点
- **删掉"你知道""说实话""其实"等填充词**——除非是角色的口癖
- 从小说改编剧本时，对白必须重写而非照搬——小说对白依赖叙述补全，剧本对白必须独立成立

#### 中国格式与好莱坞格式对照

| 元素 | 中国标准 | 好莱坞标准 |
|------|---------|-----------|
| 场景标题 | `1. 咖啡馆内 - 日` | `INT. COFFEE SHOP - DAY` |
| 角色名 | 居中加粗 | 居中大写 |
| 表演提示 | （括号内） | (parenthetical) |
| 动作段 | 正文段落 | ACTION LINE |
| 画外音 | 角色名（画外） | CHARACTER (V.O.) |
| 闪回 | 场景标题注明"闪回" | FLASHBACK - INT. ... |
| 字幕 | 【字幕：三个月后】 | TITLE CARD: THREE MONTHS LATER |

### 1.2 好莱坞标准格式（INT/EXT）

```
INT. COFFEE SHOP - DAY

A crowded downtown café. Steam rises from espresso machines.

LI MING (30s, sharp eyes, cheap suit) sits alone, watching the door.

The bell chimes. ZHANG WEI enters -- all confidence and expensive cologne.

LI MING
(under his breath)
Here we go.

ZHANG WEI spots him, crosses over. Slides into the seat uninvited.

ZHANG WEI
You look terrible.

LI MING
You look like you're trying too hard.
```

要点：
- 场景标题：`INT./EXT. 地点 - 时间`
- 角色名首次出场时在动作段用大写
- 角色名在对白前用大写
- 一页 ≈ 1 分钟银幕时间
- 电影剧本 90-120 页

### 1.3 微短剧/竖屏格式

微短剧没有统一的工业标准，但行业惯例：
- 每集 1-3 分钟，约 500-1000 字剧本
- 场景极简（通常 1-3 个场景/集）
- 对白节奏快，信息密度高
- 每集结尾必须有强钩子（悬念/反转/危机）
- 考虑竖屏构图：人物多用中近景，避免大全景

## 二、故事结构体系

### 2.1 三幕式结构（电影基础）

```
第一幕（25%）          第二幕（50%）              第三幕（25%）
建置                   对抗                        解决
│                      │                           │
开场                   游戏时间                    高潮
│                      │                           │
激励事件               中点反转                    结局
│                      │                           │
第一幕转折 ─────────→ 至暗时刻 ──────────────→ 终场画面
```

| 节点 | 位置 | 功能 |
|------|------|------|
| 开场 | 第1-3页 | 建立常态世界、主角日常 |
| 激励事件 | 约第10-12页 | 打破常态，给主角一个必须回应的问题 |
| 第一幕转折 | 约第25-30页 | 主角被迫进入新世界/新处境 |
| 游戏时间 | 第30-55页 | 主角尝试解决问题，遭遇障碍 |
| 中点 | 约第55-60页 | 假胜利或假失败，故事转向 |
| 至暗时刻 | 约第80-85页 | 主角失去一切，最低谷 |
| 高潮 | 第85-110页 | 主角用新认知/新能力解决核心冲突 |
| 终场画面 | 最后1-2页 | 与开场形成对照，展示变化 |

### 2.2 序列式结构（8序列法）

将电影分为 8 个序列，每个 12-15 分钟：

| 序列 | 时长 | 功能 |
|------|------|------|
| 序列1 | 0-12分钟 | 建置：常态世界 + 主角 + 激励事件 |
| 序列2 | 12-25分钟 | 困境：主角尝试旧方法，失败 |
| 序列3 | 25-37分钟 | 进入新世界：第一幕转折 + 新规则 |
| 序列4 | 37-50分钟 | 升级：冲突加剧，盟友/敌人明确 |
| 序列5 | 50-62分钟 | 中点：假胜利或假失败 |
| 序列6 | 62-75分钟 | 反击：主角主动出击 |
| 序列7 | 75-88分钟 | 至暗时刻：失去一切 |
| 序列8 | 88-100分钟 | 高潮 + 结局 |

### 2.3 电视剧结构

#### 单集结构（42分钟/60分钟剧集）
```
冷开场（Cold Open）: 2-3分钟
  └─ 钩子：悬念/冲突/奇观，抓住观众

第一幕（约10分钟）
  └─ A故事建立 + B故事建立

第二幕（约10分钟）
  └─ A故事升级 + B故事推进

第三幕（约10分钟）
  └─ A故事转折 + B故事交叉

第四幕（约8分钟）
  └─ A/B故事汇合或对撞

第五幕（约5分钟）
  └─ 高潮 + 集尾钩子（Tag/Coda）

片尾（Tag）: 1-2分钟
  └─ 余韵/悬念/下一集预告
```

#### A/B/C 故事线
- **A故事**：本集主线，必须有完整弧光（开始→发展→解决或悬置）
- **B故事**：副线，通常与A故事有主题呼应
- **C故事**：情感线/喜剧线/角色成长线，调节节奏

#### 季播剧结构
```
季线弧光（Season Arc）
├── 第1集：建立本季核心冲突
├── 第2-3集：扩展世界，引入新角色/新势力
├── 第4-6集：冲突升级，盟友与敌人分化
├── 第7-8集：中点反转，假胜利或假失败
├── 第9-10集：至暗时刻，核心关系破裂
├── 第11-12集：高潮 + 季终悬念
│
每集独立弧光（Episode Arc）
└── 每集有自己的 A 故事开始→发展→解决（或悬置）
```

#### 中国长剧集特殊结构
- 40集以上的长剧需要"卷"的概念（每 8-10 集为一卷）
- 每卷有自己的核心冲突和高潮
- 卷与卷之间用"大钩子"连接
- 避免中段塌腰（第 15-25 集是最容易流失观众的区间）

### 2.4 微短剧结构

```
每集 1-3 分钟，结构极简：

第1秒-10秒：钩子（必须在 10 秒内抓住注意力）
10秒-90秒：核心冲突 + 推进
最后10秒：强钩子（悬念/反转/危机/打脸）
```

- 每集必须有至少一个"爽点"或"悬念点"
- 前 3 集决定留存率（等同于网文的"黄金三章"）
- 节奏比逻辑重要——先抓住情绪，再补合理性

### 2.5 特鲁比 22 步结构

> 完整指南详见 `references/screenplay/truby-22-steps.md`

约翰·特鲁比（John Truby）的 22 步结构将故事视为"主角从虚假自我走向真实自我的道德与心理蜕变之旅"。与三幕式侧重"外部事件节点"不同，特鲁比的每一步都聚焦于主角的**内在认知变化**。

#### 22 步速览

| 阶段 | 步骤 | 核心功能 |
|------|------|---------|
| **起点** | 1. 虚假自我/幽灵 | 主角活在过去的创伤塑造的防御机制中 |
| **目标** | 2. 欲望 | 外在目标浮出水面 |
| **冲突** | 3. 对手 | 与主角争夺同一目标的对立力量 |
| **行动** | 4. 计划 | 主角采取的策略（旧→新→蜕变） |
| **伙伴** | 12. 盟友/助手 | 弥补主角缺陷、有独立弧光的角色 |
| **挫折** | 15. 表面失败 | 主角失去最珍视的东西 |
| **认知** | 16-17. 二/三次揭示 | 层层递进的自我认知 |
| **考验** | 18. 关隘/黑暗时刻 | 最后的门槛 |
| **高潮** | 19. 对决 | 物理+价值观的双重碰撞 |
| **蜕变** | 20. 自我揭示 | 主角完成内在转变 |
| **证明** | 21. 道德抉择 | 用行动而非语言证明改变 |
| **终点** | 22. 新平衡 | 与开场形成对照的新世界 |

#### 特鲁比 vs 三幕式

| 维度 | 三幕式 | 特鲁比 22 步 |
|------|--------|-------------|
| 关注点 | 外部事件节点 | 内在认知变化 |
| 转折机制 | 激励事件/中点/至暗时刻 | 三次递进的揭示 |
| 对手设计 | 阻碍者 | "另一种答案"——有独立逻辑的对立面 |
| 盟友功能 | 辅助者 | 有自己弧光的独立角色 |
| 高潮本质 | 物理冲突的解决 | 物理+道德+心理的三重碰撞 |
| 适用场景 | 节奏把控、编剧入门 | 人物深度、主题深化 |

#### 特鲁比 vs 英雄之旅

- **英雄之旅**是"外在旅程"的模板（主角去冒险，带着收获回来）
- **特鲁比**是"内在蜕变"的蓝图（主角从虚假自我走向真实自我）
- 英雄之旅的"导师"在特鲁比中变为"盟友"（盟友有自己的弧光）
- 英雄之旅的"万灵药"在特鲁比中变为"道德抉择"（用行动证明改变）

#### 何时使用特鲁比 22 步

- 需要深度人物弧光的作品（角色驱动型故事）
- 需要主题深度的作品（探讨人性、道德、成长）
- 对手需要立体化的作品（避免脸谱化反派）
- 多卷/多季长篇（每卷可运行"微型 22 步"）

#### 快速应用：7 个核心结构点

```
1. 虚假自我 → 2. 欲望 → 3. 对手 → 4. 计划
                              ↓
              15. 表面失败 → 16-17. 递进揭示
                              ↓
              19. 对决 → 20. 自我揭示 → 21. 道德抉择 → 22. 新平衡
```

## 三、人物设计（剧本专用）

### 3.1 角色功能分类

| 功能 | 说明 | 注意 |
|------|------|------|
| 主角（Protagonist） | 有明确目标，观众能共情 | 必须有缺陷，完美主角没有戏剧性 |
| 对手（Antagonist） | 与主角目标对立 | 必须有自己的合理性，不是纯粹的坏人 |
| 盟友（Ally） | 支持主角 | 需要自己的动机，不能只是附属品 |
| 导师（Mentor） | 提供关键帮助或智慧 | 通常在第二幕退场，逼主角独立 |
| 变化者（Shapeshifter） | 立场摇摆 | 增加不确定性，观众猜不透 |
| 信使（Herald） | 带来关键信息或召唤 | 触发激励事件的人 |
| 守门人（Threshold Guardian） | 阻挡主角进入新世界 | 测试主角的决心 |
| 阴影（Shadow） | 主角的黑暗面 | 最好的反派是主角可能成为的人 |

### 3.2 角色简介模板（影视专用）

```
姓名：
年龄：（选角年龄范围）
职业：
外貌特征（选角参考）：
性格关键词（3个）：
核心欲望（Want）：表层目标
核心需求（Need）：深层需求（通常与欲望矛盾）
最大恐惧：
说话风格：
标志性动作/习惯：
秘密：
与主角的关系：
角色功能：
弧光：从___变成___
```

### 3.3 角色弧光类型

| 类型 | 说明 | 经典案例 |
|------|------|---------|
| 正向弧光 | 主角克服缺陷，成长 | 《肖申克的救赎》安迪 |
| 负向弧光 | 主角堕落或毁灭 | 《教父》迈克尔 |
| 平弧光 | 主角不改变，但改变周围的世界 | 《老人与海》圣地亚哥 |
| 循环弧光 | 回到原点，但观众理解了为什么 | 《恐怖游轮》 |

### 3.4 群像戏人物管理

- 核心角色不超过 5 个（观众记忆上限）
- 每个角色必须有独特的"识别标签"（外貌/口癖/行为模式）
- 用"关系图"管理角色之间的动态关系
- 群像戏需要"锚点角色"——观众通过他的视角理解其他角色

## 四、场景设计

> 详见 `references/screenplay/scene-construction.md`

### 4.1 场景的戏剧功能

每一场戏必须至少完成以下之一：
1. **推进剧情**：发生了一件改变局面的事
2. **揭示人物**：通过选择/反应展示角色本质
3. **传递信息**：观众需要知道的关键信息
4. **制造转折**：状态改变（好→坏，或坏→好）
5. **建立关系**：角色之间的力量动态变化

如果一场戏不完成以上任何一项——删掉它。

### 4.2 场景节拍

每个场景内部有多个"节拍"（beat），每个节拍是一次情绪或力量的转换：

```
场景：咖啡馆谈判

节拍1：李明坐下，表面平静（力量：李明）
节拍2：张伟拿出照片，占据上风（力量：张伟）
节拍3：李明反问一个问题，张伟犹豫（力量：李明）
节拍4：张伟亮出底牌，李明震惊（力量：张伟）
节拍5：李明说出一个秘密，局面逆转（力量：李明）
```

### 4.3 场景长度控制

| 类型 | 建议长度 | 说明 |
|------|---------|------|
| 过渡场景 | 半页-1页 | 快进快出 |
| 对话场景 | 2-4页 | 需要冲突支撑 |
| 重要冲突 | 3-6页 | 可以长，但每个节拍都要有变化 |
| 动作场景 | 灵活 | 取决于复杂度 |
| 单场景上限 | 5-7页 | 超过需要充分理由 |

### 4.4 转场技巧

| 技巧 | 效果 | 适用 |
|------|------|------|
| 直切（CUT） | 节奏快 | 紧张段落、时间连续 |
| 溶解（DISSOLVE） | 时间流逝 | 时间跳跃 |
| 叠化 | 情绪延续 | 情感连接 |
| 跳切（JUMP CUT） | 省略过程 | 强调结果、焦虑感 |
| 声音桥接 | 用声音连接两个场景 | 情绪延续、信息传递 |
| 匹配剪辑 | 视觉相似连接 | 主题呼应 |
| 黑屏/白屏 | 段落分隔 | 章节感 |

### 4.5 鸿沟驱动的场景设计

> 完整理论详见 `references/screenplay/mckee-gap-theory.md`

**鸿沟（The Gap）** 是麦基《故事》的核心概念之一：角色采取行动后期望获得某种反应，但实际得到的反应与期望之间存在落差。这个落差就是"鸿沟"，它是驱动场景价值转变的核心引擎。

#### 4.5.1 鸿沟与场景价值转变

每场戏的核心是价值状态的正负转变。鸿沟是推动这种转变的机制：

```
角色行动（期望反应 X）→ 实际反应 Y → 鸿沟 = X 与 Y 的落差
  ↓
角色被迫重新评估 → 调整策略/暴露真相/改变关系
  ↓
场景价值状态发生转变
```

#### 4.5.2 鸿沟的三个层次

| 层次 | 落差幅度 | 角色反应 | 叙事功能 | 出现频率 |
|------|---------|---------|---------|---------|
| 最小鸿沟 | 微小意外 | 微调策略 | 维持节奏、积累张力 | 高（每场1-3个） |
| 中度鸿沟 | 明显偏差 | 改变路线 | 推动情节转折 | 中（关键场景） |
| 最大鸿沟 | 根本颠覆 | 重塑世界观 | 故事重大转折 | 低（每幕1-2个） |

#### 4.5.3 鸿沟标注的节拍设计

在节拍模板中加入鸿沟标注，使场景设计更加精准：

```
节拍N：[状态] → [角色行动] → [期望反应] → [实际反应（鸿沟）]
  └─ 鸿沟深度：最小/中度/最大
  └─ 角色发现：___（外部事实/他人真相/自我真相）
  └─ 力量转移方向：从___到___
```

#### 4.5.4 鸿沟与体裁

| 体裁 | 核心鸿沟模式 | 说明 |
|------|-------------|------|
| 网文 | "打脸"鸿沟 | 反派低估主角，遭遇认知落差，读者获得爽感 |
| 悬疑 | "真相鸿沟" | 侦探/读者对真相的认知与实际真相的落差 |
| 言情 | "情感鸿沟" | 角色对彼此情感状态的误解与真相的落差 |
| 奇幻/武侠 | "力量鸿沟" | 角色对自身/对手实力的误判 |
| 喜剧 | "期望鸿沟" | 角色的严肃期望与荒谬现实的落差 |

#### 4.5.5 鸿沟设计自检

- [ ] 每个场景至少包含一个鸿沟
- [ ] 鸿沟推动了价值状态的转变
- [ ] 鸿沟深度与场景的戏剧功能匹配
- [ ] 鸿沟类型在场景序列中有变化（不总是同一种落差）
- [ ] 全剧的鸿沟深度逐步递增，高潮处为最大鸿沟

## 五、对白写作

> 详见 `references/screenplay/dialogue-mastery.md`

### 5.1 对白的五层功能

1. **表层意思**：角色说出来的话
2. **潜台词**：角色真正想表达的
3. **人物塑造**：通过说话方式展示性格
4. **情节推进**：传递关键信息
5. **节奏控制**：长句/短句/沉默的交替

### 5.2 对白基本原则

- **每句话都要有目的**——如果删掉这句对白故事不受影响，就删掉
- **角色不是在交流信息，是在争取什么**——每段对话至少有一方在"要"什么
- **沉默比语言更有力**——角色不说话的时候，观众在读他的表情
- **打断和抢话是常态**——真实对话中很少有人等对方说完
- **角色只说自己会说的话**——教授和工人不会用同一套词汇

### 5.3 避免的对白问题

| 问题 | 错误示例 | 正确做法 |
|------|---------|---------|
| 信息播报 | "你好，我是张伟，我是这家公司的新来的工程师" | 通过动作和反应传递信息 |
| 解释性叙述 | "他这么说是因为他从小缺少父爱" | 用行为暗示，让观众自己理解 |
| 全员书面语 | 所有角色都像在念论文 | 每人有独特的口语特征 |
| 对白后解释 | 对白后紧跟"他真正的意思是……" | 删掉解释，信任观众 |
| 问答式对话 | A问B答，B问A答 | 加入打断、沉默、答非所问 |

### 5.4 对白风格光谱

```
极简 ←─────────────────────────────────→ 极繁
│                                              │
│  科恩兄弟          昆汀          韦斯·安德森  │
│  (少即是多)        (话痨美学)     (精准矫饰)  │
│                                              │
│  中国参考：                                     │
│  刁亦男             娄烨          姜文         │
│  (冷硬克制)        (生活流)       (话痨+隐喻)  │
```

### 6.2 Logline 公式


## 八、改稿与应对 Notes

> 详见 `references/screenplay/development-docs.md` 第五节

### 8.1 改稿心态

- 制片人给 note 不是在否定你，是在告诉你"这里观众可能会有这个问题"
- 先理解 note 背后的真实问题，再决定怎么改
- 保护故事核心（theme + 主角弧光），其他都可以动

### 8.2 常见 Notes 与应对

| Note 类型 | 真实问题 | 应对策略 |
|----------|---------|---------|
| "节奏太慢" | 信息密度不够或冲突不够 | 合并场景、删过渡、提前冲突 |
| "角色不讨喜" | 缺陷太重或缺乏共情点 | 加一个"救猫咪"场景或展示合理动机 |
| "看不懂" | 信息传递不清 | 补一个"锚点场景"或简化线索 |
| "太平了" | 缺少转折或意外 | 在第二幕中点加反转 |
| "结尾不满意" | 高潮不够有力或与主题脱节 | 检查高潮是否回应了主角的核心需求 |
| "投资太大" | 制作成本超预算 | 合并场景、减少外景、压缩群演 |

### 8.3 改稿流程

```
第一稿：把故事讲完（结构+情节）
  ↓
第二稿：把故事讲对（逻辑+节奏）
  ↓
第三稿：把故事讲好（对白+细节）
  ↓
第四稿：把故事讲精（删冗余+调节奏）
  ↓
定稿：制片人/导演确认后锁定
```

## 十、中国影视行业认知

### 10.1 平台特征

| 平台 | 偏好 | 注意 |
|------|------|------|
| 爱奇艺 | 悬疑、品质剧 | 迷雾剧场标准高 |
| 优酷 | 古装、都市 | 大IP偏好 |
| 腾讯视频 | 全品类 | 生态链完整 |
| 芒果TV | 甜宠、青春 | 女性向为主 |
| 抖音/快手 | 微短剧 | 节奏极快，竖屏 |
| B站 | 青年向、创新 | 弹幕文化 |

### 10.2 审查红线（编剧必知）

- 不能正面表现犯罪手法的细节
- 不能美化毒品、赌博
- 历史题材不能戏说重大历史事件
- 涉案题材需要"正义终将到来"的结局
- 鬼怪类需要合理化解释（幻觉/科学/梦境）
- 暴力尺度：可以暗示，不宜过度血腥
- 情感尺度：可以暧昧，不宜过度露骨

### 10.3 行业流程

```
创意提案 → 平台/制片公司评估 → 签约 → 开发（大纲+分集） → 审核 → 写剧本 → 审核 → 筹备 → 拍摄 → 后期 → 上线
```

编剧在每个阶段的交付物不同，需要根据阶段调整工作方式。

## 十一、编剧工具箱

### 11.1 自检清单（每场戏）

- [ ] 这场戏的戏剧功能是什么？（推进/揭示/转折/信息/关系）
- [ ] 场景开始和结束时，状态是否不同？
- [ ] 是否有至少一个冲突或张力点？
- [ ] 对白是否在争取什么，而非只是交流？
- [ ] 是否可以用更短的篇幅达到同样效果？
- [ ] 这场戏是否可以删掉而不影响故事？

### 11.2 自检清单（整本剧本）

- [ ] Logline 能否在 10 秒内说清故事？
- [ ] 主角是否有清晰的目标和缺陷？
- [ ] 每一幕的转折是否有力？
- [ ] 中点是否有真正的反转？
- [ ] 高潮是否回应了主角的核心需求？
- [ ] 结局是否与开场形成对照？
- [ ] 是否有"死场景"（不推进任何功能的场景）？
- [ ] 对白是否有个人风格差异？
- [ ] 制作成本是否在合理范围内？

### 11.3 常用叙事技巧

| 技巧 | 说明 | 示例 |
|------|------|------|
| 救猫咪 | 开头展示主角的善意，建立共情 | 主角救了一只猫，观众立刻喜欢他 |
| 麦高芬 | 驱动情节但本身不重要的物件 | 《低俗小说》的箱子 |
| 契诃夫之枪 | 出现的元素必须被使用 | 第一幕出现的枪，第三幕必须开火 |
| 戏剧反讽 | 观众知道角色不知道的信息 | 观众知道杀手在衣柜里，角色不知道 |
| 悬念 vs 惊吓 | 悬念是"要发生"，惊吓是"突然发生" | 希区柯克的炸弹理论 |
| 红鲱鱼 | 故意误导观众的假线索 | 悬疑片中的假嫌疑人 |
| 麦克格芬 | 观众和角色都在追逐但最终不重要的东西 | 推动情节的工具 |
| Deux ex machina | 外力突然解决问题（应避免） | 突然出现的救援 |

## 十二、写作铁律（编剧版）

### 必须做
- 每一场戏都要有目的
- 角色通过选择揭示本质
- 对白要有潜台词
- 展示（Show），不要告诉（Tell）
- 尊重观众的智商

### 禁止做
- 角色突然变聪明或变蠢来服务情节
- 用巧合解决核心冲突
- 解释性对白（"正如你所知……"）
- 无功能的场景（纯粹凑时长）
- 所有角色用同一种说话方式
- 结尾突然升华主题（主题应该贯穿始终）
