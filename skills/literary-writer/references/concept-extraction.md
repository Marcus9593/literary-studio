# 概念提炼指南

## 一、What-If 方法论

### 1.1 核心公式

```
What if [角色] 在 [情境] 下，发现/遭遇 [转折]？
```

### 1.2 从碎片到 What-If

用户的输入通常是碎片化的：

| 用户输入 | What-If 转化 |
|----------|-------------|
| "我想写修仙" | What if 一个普通人发现自己能修仙，但每次突破都要付出不可逆的代价？ |
| "我想写末日" | What if 末日来临，但只有一个人知道末日的真正原因，而没人相信他？ |
| "我想写爱情" | What if 两个相爱的人发现，他们的相遇是一场被精心设计的实验？ |
| "我想写悬疑" | What if 一个侦探发现，他追查的凶手就是未来的自己？ |

### 1.3 What-If 质量检查

好的 What-If 应该满足：
- [ ] 包含角色（谁）
- [ ] 包含情境（在哪/什么背景）
- [ ] 包含转折（发生了什么意外）
- [ ] 有冲突潜力（能驱动一整本书的情节）
- [ ] 有情感共鸣（读者会关心这个角色的命运）

### 1.4 多方案碰撞

当用户的想法模糊时，生成 2-3 个 What-If 候选：
- 选择最有冲突潜力的
- 选择最能引起情感共鸣的
- 选择最独特的（避免套路）

## 二、主题提取

### 2.1 什么是主题

主题是故事探讨的**永恒问题**，不是情节描述。

| 好的主题 | 不好的主题 |
|----------|-----------|
| 权力是否必然腐蚀人心 | 主角打败大反派 |
| 自由与安全的永恒矛盾 | 主角穿越到异世界 |
| 真相是否 always 值得追求 | 主角破获连环杀人案 |
| 个人幸福与集体责任的冲突 | 主角和女主在一起了 |

### 2.2 主题提取规则

1. **主题是问题，不是答案**——故事探讨问题，不给出标准答案
2. **最多 2 个主题**——超过 2 个会导致故事散焦
3. **主题要与 What-If 呼应**——What-If 是主题的具体化

### 2.3 从 What-If 提取主题

```
What if 一个普通人能修仙，但每次突破都要付出不可逆的代价？

主题：长生是否值得用人性去交换？
```

```
What if 末日来临，只有一个人知道真相，但没人相信他？

主题：当真相与希望冲突时，应该选择哪一个？
```

## 三、核心冲突推导

### 3.1 好的冲突是"两难"

| 类型 | 示例 | 质量 |
|------|------|------|
| 善恶分明 | 主角 vs 邪恶势力 | 差——没有真正的选择 |
| 两难 | 主角必须在救一个人和救一百个人之间选择 | 好——两个选择都有代价 |
| 矛盾统一 | 主角越强大，越失去人性 | 好——目标本身带来代价 |

### 3.2 冲突层级

```
内心冲突 → 人际冲突 → 社会冲突 → 命运冲突
```

最好的故事同时在多个层级有冲突。

### 3.3 从主题推导冲突

```
主题：长生是否值得用人性去交换？

核心冲突：主角渴望长生，但每次突破都在失去人性。
  - 内心冲突：想要变强 vs 不想失去人性
  - 人际冲突：变强后与旧友的疏离
  - 命运冲突：最终发现长生的代价是成为自己最讨厌的存在
```

## 四、前提陈述（Premise）

### 4.1 公式

```
当[主角]面对[核心冲突]时，[关键选择]导致[结果]，揭示了[主题]。
```

### 4.2 示例

```
当韩立面对"修仙需要牺牲人性"的现实时，
他选择用最冷酷的方式生存，
最终成为最强但也最孤独的修士，
揭示了"长生是否值得"的主题。
```

```
当一个侦探发现追查的凶手就是未来的自己时，
他必须决定是阻止未来的自己还是接受命运，
最终选择改变命运但付出巨大代价，
揭示了"自由意志是否存在"的主题。
```

### 4.3 Premise 检查清单

- [ ] 包含主角身份
- [ ] 包含核心冲突
- [ ] 包含关键选择
- [ ] 包含结果
- [ ] 包含主题
- [ ] 一句话能说完
- [ ] 让人想看这个故事

## 五、各体裁适配

### 5.1 网文

What-If 要包含"爽点"维度：
- 主角有什么优势/金手指？
- 读者爽感来自哪里？
- 升级/打脸/逆袭的节奏是什么？

### 5.2 纯文学

主题可以更深、更开放：
- 不一定要有明确答案
- 可以探讨更抽象的问题
- 人物弧光比情节更重要

### 5.3 剧本

冲突要更视觉化、更外化：
- 冲突必须能"看到"
- 对白比内心独白更重要
- 场景转换要有视觉冲击

### 5.4 短篇

只提炼一个核心 What-If：
- 不要有多条线
- 聚焦一个冲突
- 留白比展开更重要

```json
  "what_if": "如果一个普通人发现自己能修仙，但每次突破都要付出不可逆的代价？",
  "themes": ["长生是否值得用人性去交换"],
  "core_conflict": "主角渴望变强，但每次突破都在失去人性",
  "premise": "当一个普通人发现自己能修仙时，他选择追求长生，但每次突破都在失去人性，最终成为最强但也最孤独的修士，揭示了长生是否值得的主题。"
}
```

此 JSON 会传递给 webnovel-init 的 Step 2 使用。
