# 网文创作指南（完整版）

## 核心原则

网文的本质是"连载型娱乐内容"，读者追求的是：
- **爽感**：主角成长、打脸反派、获得认可
- **期待感**：接下来会发生什么？
- **代入感**：读者能与主角共情
- **节奏感**：读起来不累，停不下来

---

## 一、黄金三章法则

前三章（约6000-9000字）决定读者是否继续阅读：

### 第一章：钩子
- **前500字**：必须有吸引力（冲突、悬念、奇观、反转）
- **主角出场**：快速建立代入感
- **核心设定**：让读者知道这是什么类型的故事
- **章末悬念**：让读者想点"下一章"

### 第二章：展开
- 展示金手指/特殊能力/机遇
- 第一个小冲突
- 强化期待感

### 第三章：定调
- 确立故事的核心爽点
- 第一次小高潮
- 让读者明确"这本书能给我什么"

---

## 二、写作铁律（三大定律）

| 定律 | 规则 | 检查方式 |
|------|------|----------|
| **大纲即法律** | 严格执行大纲，不得擅自发挥 | 审查时对照大纲 |
| **设定即物理** | 实力/招式/物品 ≤ 已有记录 | 写作前查询确认 |
| **发明需识别** | 新实体需在章节完成后提取归档 | 章节完成后处理 |

### 章节约束分层

#### Hard（必须）
- 本章可读性达标：读者能回答"发生了什么 / 谁在做什么 / 为什么"
- 本章必须存在清晰推进：问题、目标、代价、关系变化、信息变化至少一项
- 若上章有明确承诺（钩子/未闭合问题），本章必须回应
- 禁止输出占位正文

#### Soft（建议）
- 开头尽早进入冲突/风险/强情绪（建议前 200-400 字）
- 未闭合问题或下一章期待锚点建议放在章末或后段
- 局面变化建议保持节奏感（参考 800-1400 字一个脉冲）
- 微兑现频率按题材建议执行

#### Style（可选强化）
- 对话尽量带意图（试探/回避/施压/诱导），减少纯说明句
- 避免连续大段纯解释；若必须解释，建议切分为"信息 + 行动/反应"
- 避免"回去休息了"式机械收尾

---

## 三、Anti-AI 写作对抗

### 词汇层
- 删万能副词（缓缓/淡淡/微微），换具体动作
- 避免高频 AI 词汇密集出现（"眸中闪过""瞳孔微缩"等）
- 500 字内相同模板不超过 3 次

### 句式层
- 禁止"起因→经过→结果→感悟"四段闭环
- 避免连续同构句（≥3 句主谓宾结构一致）
- 不要每段都以总结句收尾
- 避免同一信息用不同句式重复 2-3 遍

### 叙事层
- 节奏不能匀速——段落信息密度要有快慢变化
- 避免"他不知道的是……""殊不知……"戏剧性反讽提示
- 章末禁止"安全着陆"——冲突不能完美解决，必须留不安感
- 展示后不解释——先用动作展示，不要紧接着一句话解释含义

### 情感层
- 情绪描写禁止标签化（"他感到愤怒""她非常紧张"）
- 用行为/生理暗示替代：手虚握符箓、进门前神识扫一圈
- 避免情绪即时切换（上句愤怒下句平静，无过渡）
- 不要所有角色用同一套反应模板

### 对话层
- 对话必须带意图冲突，禁止纯信息宣讲
- 每人带各自心思说话，有抢话、沉默、答非所问
- 不要全员书面语，要有口语特征和个人口癖
- 对白后不要跟解释性叙述

## 四、网文类型与题材配置

### 玄幻/仙侠
- **升级体系**：练气→筑基→金丹→元婴→化神→...（清晰的等级系统）
- **金手指**：系统、老爷爷、神器、血脉、重生记忆
- **核心爽点**：升级突破、越级挑战、宝物获取、势力扩张
- **节奏**：快速升级→遇瓶颈→突破→新的挑战
- **境界突破是核心期待**，阶位制建议可视化（8-10级体系）
- **资源货币化体系**（灵石/丹药/功法）是核心微兑现载体
- **过渡章容忍度**：允许更多铺垫章（连续不超过3章）

### 都市
- **重生/系统**：获得前世记忆或特殊系统
- **核心爽点**：赚钱、打脸、逆袭、商战胜利
- **节奏**：低调→展示→震惊→更大的展示
- **装逼打脸系列是核心爽点**
- **现代背景要求身份隐藏→掉马的节奏控制**
- **3章一峰节奏**：第1章困境，第2章能力初展，第3章小胜+新阻力

### 言情/甜宠
- **核心**：情感互动，关系推进，心动与虐心交织
- **情绪钩是王牌**（心疼/心动/吃醋）
- **关系进展是最重要的微兑现**
- **感情线断档容忍度极低**（不超过5章）

### 游戏/电竞
- **游戏设定**：职业、技能、装备、公会
- **核心爽点**：操作碾压、首杀、战术胜利、装备获取
- **节奏**：新手村→公会→比赛→世界级赛事
- **数值反馈建议可视化**
- **金手指建议设置上限/消耗/冷却**

### 无限流
- **副本设计**：每个副本是一个独立的小故事
- **核心爽点**：通关、获取能力、团队配合
- **节奏**：进入副本→探索→危机→通关→休整→下一个副本

### 规则怪谈
- **紧张氛围要求高钩子强度**
- **过渡章容忍度极低**（1章）
- **恐怖感来自规则和代价，而非纯氛围堆叠**

### 悬疑/推理
- **逻辑完整性优先于爽点密度**
- **信息兑现是核心微兑现**
- **真相揭示是王牌爽点**

### 知乎短篇
- **短平快，强反转，情绪冲击**
- **过渡章窗口极窄**，每章至少有一项可感知收获
- **极高钩子强度要求**

### 历史穿越
- **知识优势 > 武力优势**，推导过程要展示
- **3章一峰节奏**：第1章困境，第2章知识初展，第3章小胜+新阻力
- **历史有惯性**，改变一件事会引发连锁反应

### 替身文/虐文
- **情绪钩是绝对核心**（虐心→心疼→期待）
- **身份掉马是王牌爽点**
- **感情线断档容忍度极低**

## 五、爽点工程

### 六种爽点执行模式

#### 1. 装逼打脸
```
对方轻视 → 主角展示实力 → 对方震惊/后悔
```
适用：都市、玄幻、职场

#### 2. 扮猪吃虎
```
表面弱小 → 关键时刻爆发 → 众人惊艳
```
适用：重生、隐藏身份、低调流

#### 3. 越级反杀
```
实力差距明显 → 主角逆袭 → 敌人不可置信
```
适用：玄幻、武侠、竞技

#### 4. 打脸权威
```
权威质疑 → 主角用实力证明 → 权威认可/尴尬
```
适用：职场、学院、技术流

#### 5. 反派翻车
```
反派得意 → 计划破产 → 反派狼狈
```
适用：所有题材

#### 6. 甜蜜超预期
```
平淡日常 → 意外惊喜 → 情感升温
```
适用：言情、甜宠

### 爽点三段式结构 (30/40/30)

#### 铺垫阶段 (30%篇幅)
- **建立预期**：读者应该期待什么
- **制造反差**：当前状态 vs 即将展现的状态
- **信息差设置**：读者知道什么？角色知道什么？

#### 兑现阶段 (40%篇幅)
- **触发时机**：什么事件触发爽点
- **展现方式**：用对话/动作/结果展现
- **情绪高峰**：爽点的最高潮瞬间

#### 微反转阶段 (30%篇幅)
- **假结束**：读者以为爽点结束了
- **还有一手**：其实还有更厉害的
- **余韵**：微反转后的状态与暗示

### 打脸四步法

**Step 1 铺垫**：提前1-2章建立信息差（读者知道主角底牌，反派不知道）

**Step 2 挑衅**：通过1-3次挑衅建立对照组（反派被追捧 vs 主角被贬低）

**Step 3 拉扯**：2-3轮交锋，主角示弱 → 反派得意 → 期待拉满

**Step 4 爆发**：物理碾压 + 精神打击 + 围观群众反应 + 实质收获

### 爽点密度建议

| 周期 | 要求 |
|------|------|
| 逐章 | 优先保证有"爽点或同等兑现"，允许过渡章低密度 |
| 每 5 章 | 建议 ≥1 个组合爽点（2种模式叠加） |
| 每 10-15 章 | 建议 ≥1 个里程碑爽点（改变主角地位） |

### 爽点升级路径

**维度1 规模升级**：打脸小流氓 → 打脸世家少爷 → 打脸宗门长老 → 覆灭一个帝国

**维度2 方式升级**：暴力碾压 → 智谋碾压 → 规则掌控 → 因果抹杀

**维度3 受众升级**：让邻居震惊 → 让全城震惊 → 让全国震惊 → 让诸天神魔战栗

### 微反转类型

- **还有更强**：展示了一手，其实还有更厉害的
- **意外收获**：本来只想完成目标，结果还有意外好处
- **对方更惨**：对方以为只是小失败，其实是大翻车
- **情感升温**：本以为只是甜，其实更甜
- **真相揭示**：表面是A，其实是B，而且B更厉害

## 六、信息差设计

### 信息差层级
| 层级 | 说明 |
|------|------|
| 读者已知 | 读者通过前文已经知道的信息 |
| 角色已知 | 场景中的角色知道的信息 |
| 信息差核心 | 读者知道但角色不知道的关键信息 |
| 反转信息 | 微反转时揭示的新信息 |

### 信息差运用
```
读者知道主角是隐藏大佬 + 角色不知道 = 期待感
角色嘲讽主角 + 读者知道主角实力 = 爽感积累
主角展示实力 + 角色震惊 = 爽感释放
```

---

## 七、压扬比例控制

| 题材类型 | 压扬比例 | 说明 |
|---------|---------|------|
| 传统爽文 | 压3扬7 | 轻度压迫，快速释放 |
| 硬核正剧 | 压5扬5 | 平衡叙事 |
| 虐恋/黑深残 | 压7扬3 | 长期压抑，爆发更爽 |

### 压扬节奏控制
```
慢铺垫（压） → 快爆发（扬） → 停顿 → 再加速
```

### 句式节奏
- **短句**：快节奏，动作密集
- **长句**：慢节奏，细节描写
- **交替**：制造节奏变化

---

## 八、章节结构

### 标准章节（2000-3000字）
```
开头（200字）：承接上章 / 场景建立
发展（1500字）：核心事件展开
高潮/转折（500字）：本章最重要的时刻
章末钩子（200字）：悬念、反转、或暗示
```

### 章末钩子类型
1. **悬念型**："门缓缓打开，站在门口的人让他瞳孔骤缩。"
2. **期待型**："明天的比赛，他将面对整个学院最强的对手。"
3. **反转型**："他以为自己赢了，直到看见那封信。"
4. **升级型**："丹田中，一股前所未有的力量正在苏醒。"
5. **危机型**："三道黑影从暗处走出，将他围在中间。"
6. **渴望钩**：展示即将获得的宝物/地位/认可
7. **选择钩**：主角面临两难抉择

## 九、节奏控制

### 爽点分布
- 小爽点：每1-2章一个（小胜利、小突破）
- 中爽点：每5-10章一个（打败对手、获得宝物）
- 大爽点：每20-50章一个（大境界突破、大势力覆灭）

### 避免的节奏问题
- **水字数**：无意义的对话和描写
- **升级过快**：爽感来自过程，不是结果
- **反派智商下线**：反派太弱会让胜利廉价
- **重复套路**：同样的打脸模式用三次以上读者会腻

### Strand 平衡警告

| 情节线 | 警告条件 |
|--------|----------|
| Quest（主线） | 连续 5+ 章 |
| Fire（感情线） | >10 章未出现 |
| Constellation（世界观） | >15 章未出现 |

## 十、伏笔三层级

| 层级 | 回收周期 | 示例 |
|------|----------|------|
| 核心 | 50-300章 | 主角身世/终极敌人/金手指来历 |
| 支线 | 30-100章 | 配角动机/神秘物品/师门秘密 |
| 装饰 | 10-30章 | 场景细节/小习惯/路人提及 |

## 十一、数据思维

### 读者心理
- **期待管理**：预告即将发生的精彩事件
- **情感投资**：让读者关心角色命运
- **信息差**：读者知道而角色不知道（制造紧张），或角色知道而读者不知道（制造悬念）
- **公平性**：金手指要有代价或限制

### 留存关键
- 每章结尾都有继续阅读的理由
- 定期满足读者期待（不能一直吊胃口）
- 角色成长要可见（等级、地位、实力）
- 配角不要太多，容易记不住

---

## 十二、平台规范注意

- 避免敏感政治话题
- 暴力描写适度
- 感情线注意尺度
- 注意各平台的审核标准差异
- 原创性：避免过度借鉴（洗稿/融梗）

---

## 十三、角色命名与声音

### 命名规则
- 主角名字要好记、有辨识度，避免与已有知名角色撞名
- 配角名字不要与主角太相似（容易混淆）
- 反派名字可以带暗示性（但不要脸谱化）
- 势力/组织名称要有气势或特色

### 角色声音
- 每个角色应有独特的说话方式：用词、节奏、口头禅
- 高地位角色说话简洁有力
- 反派可以有标志性台词
- 避免所有角色都用同一套书面语

---

## 十四、战斗场景写作

### 基本原则
- 战斗要有目的（不是为了打而打）
- 用具体动作替代抽象描述（"一拳击飞"而非"展现实力"）
- 节奏要快——短句、断句、动作密集
- 战斗中穿插心理活动（盘算、恐惧、决心）

### 常见问题
- ❌ "他使出了全力"——太抽象，要用具体招式和效果
- ❌ 连续3段以上的纯战斗描写——需要穿插反应和心理
- ❌ 战斗后无善后——需要战利品、认可、伤势等后果

---

## 十五、情感描写

### 核心原则
- 用生理反应+微动作替代情绪标签
- 情绪要有积累过程，不能即时切换
- 不同角色对同一事件有不同情绪反应

### 常用技巧
- 愤怒：握紧拳头、咬牙、声音发抖、太阳穴跳动
- 恐惧：后背发凉、腿软、呼吸急促、瞳孔放大
- 心动：心跳加速、脸红、不自觉微笑、目光追随
- 悲伤：喉咙发紧、眼眶发酸、声音嘶哑、沉默

---

## 十六、对话写作

### 核心原则
- 每段对话都应有立场碰撞或信息博弈
- 对话要推进情节或揭示人物，不能只是闲聊
- 有抢话、沉默、答非所问——真实对话不是你一句我一句

### 对话层次
- **表面层**：角色说出来的话
- **潜台词**：角色真正想表达的
- **信息差**：读者知道但角色不知道的（或反之）

### 常见问题
- ❌ 对话像说明会——角色在解释背景给读者听
- ❌ 所有角色说话方式一样——缺乏个性
- ❌ 对话后跟解释性叙述——"他这么说是因为……"
