# 输出格式规范

> 本文件包含各体裁的输出格式要求。按需加载，不要在每次触发时自动读取。

---

## 一、大纲输出

```
# 《作品名称》大纲

## 一句话概述
[用一句话概括整个故事]

## 核心主题
[作品探讨的核心问题]

## 人物表
### 主角：[姓名]
- 目标：
- 动机：
- 缺陷：
- 弧光：

## 章节大纲
### 第一幕：[名称]（约占25%）
#### 第1章：[章节名]
- 场景：
- 核心事件：
- 情感走向：
...
```

## 二、正文输出字数标准

| 体裁 | 每章/每集字数 | 备注 |
|------|-------------|------|
| 小说 | 2,000-5,000 字 | 按章节输出 |
| 电影剧本 | 90-120 页 | 标准剧本格式 |
| 电视剧剧本 | 每集 42-60 页 | 含冷开场+5幕+片尾 |
| 微短剧剧本 | 每集 500-1,000 字 | 1-3 分钟时长 |
| 网文 | 每章 2,000-3,000 字 | 章末设悬念钩子 |
| 图书 | 按章节/小节 | 配合案例和图表 |

## 三、格式规范

### 小说/网文
- 中文排版，段首缩进两字符，对话用引号

### 中国剧本标准
- 场景标题：`序号. 地点内/外 - 时间`
- 角色名加粗居中，对白缩进

### 好莱坞标准
- `INT./EXT. 地点 - TIME`
- 角色名大写，动作段大写首次出场

### 英文创作
- 遵循标准英文出版格式

## 四、剧本开发文档

| 文档 | 篇幅 | 说明 |
|------|------|------|
| Logline | 1-2 句话 | 概括故事核心 |
| Treatment | 10-20 页 | 散文体故事大纲 |
| Beat Sheet | 2-5 页 | 结构节拍表 |
| Series Bible | 20-50 页 | 剧集项目书（电视剧） |
| 分集大纲 | 每集 1-2 页 | 核心事件 |
| 分场大纲 | 每集 3-5 页 | 场景概述 |
