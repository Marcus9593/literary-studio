# 角色行为约束矩阵

> 项目：{title}｜创建：{date}

## 使用说明

此文件定义每个核心角色的行为边界。context-agent 在组装写作任务书时会引用这些约束；reviewer 在审查时会检查正文是否违反约束。

## 约束类型

| 类型 | 说明 | 违反后果 |
|------|------|----------|
| **hard** | 绝对不能违反的约束 | severity: critical, blocking: true |
| **soft** | 一般不会违反，但特殊情境下可以 | severity: medium, blocking: false |
| **evolving** | 随剧情发展可以改变 | 无铺垫变化 → severity: high |

## 约束字段

| 字段 | 类型 | 说明 |
|------|------|------|
| constraint_id | string | 约束 ID（如 P001、A001） |
| type | hard/soft/evolving | 约束类型 |
| category | string | 行为类别：combat / social / moral / emotional / knowledge |
| rule | string | 约束规则描述 |
| trigger_conditions | string | 触发条件（何时检查此约束） |
| exceptions | string | 例外情况（"无"表示无例外） |
| evidence | string | 设定来源（哪章/哪个设定文件确立的） |
| confidence | float | 置信度（从文本推断 vs 明确设定） |

## 填写指南

1. **从设定文件提取**：先从主角卡、女主卡、反派设计等设定文件中提取明确的行为约束
2. **从已有文本推断**：分析已写章节中角色的行为模式，推断隐含的约束
3. **标记置信度**：明确设定的约束 confidence=1.0，从文本推断的根据确定程度给 0.5-0.9
4. **定期更新**：随着剧情发展，evolving 类型的约束需要更新
5. **与 reviewer 联动**：reviewer 会自动检查此文件中的约束，确保角色行为一致
