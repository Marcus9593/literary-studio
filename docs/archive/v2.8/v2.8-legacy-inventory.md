```text
inferred + 非中英文字符占比异常      → reject
user 创建                            → 始终分配 char_*
```

## 四、Entity ID 现实 vs 法则 6

| 维度 | 现实 | 目标（已签字） |
|------|------|----------------|
| 算法 | name 作 id | **prefix + base32(8)**，如 `char_a3f9k2m1` |
| 改名安全 | 否 | `name` 可变，`aliases[]` |
| 合并 | 无 | `merged_into` |

**迁移必做：**

1. 为每个 **保留** 的 character 生成新 `char_{base32_8}`
2. 写 `legacy_id: "<旧 name-key>"`
3. 更新 relationships 若含 name 引用（需扫描）
