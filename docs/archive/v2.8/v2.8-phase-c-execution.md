# V2.8 · Phase C 执行手册

> **状态**：CTO 已批准 2C-3 · 脚本边界：迁移只做 I/O 变换，业务读写在 `story-kb/store.js` / `measurement/review-facade.js`

## 脚本（必须独立执行）

| 脚本 | 职责 |
|------|------|
| `migration-backup.mjs` | 备份 `studio.json` + `projects/{id}.tar.gz` |
| `migration-phase-c.mjs` | 幂等迁移；写 `migration/id-mapping.json` |
| `migration-validate.mjs` | 校验 + 引用完整性 |
| `migration-enable-legacy-guard.mjs` | Cleanup **前**冻结 `studio.json` 旧写入 |

共享规则库：`migration-lib.mjs`（非业务逻辑）

## 命令

```bash
cd literary-studio

node backend-node/scripts/migration-backup.mjs --all

node backend-node/scripts/migration-phase-c.mjs --all --backup-dir data/backup/phase-c-XXXX

node backend-node/scripts/migration-validate.mjs --all

# Spot check 通过后（已完成 2026-06-02）
node backend-node/scripts/migration-enable-legacy-guard.mjs
```

单项目：将 `--all` 换为 `--project <id>`。

**Cleanup A** 已合入代码（见 [v2.8-phase-d-stabilization.md](./v2.8-phase-d-stabilization.md)）。

## 幂等

重复执行 `migration-phase-c.mjs` 时，若 `knowledge/entities/characters.json` 含 `migration_version: phase-c-v1`，**直接退出**，不重新生成 `char_*`。

└── archived-health_snapshot.json  # 若曾存在

projects/{id}/knowledge/entities/
