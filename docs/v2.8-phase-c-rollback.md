# V2.8 · Phase C Rollback Plan

> **状态**：定稿 — Phase C **开始前**必须执行备份；失败时按本文恢复  
> **前置**：[v2.8-migration-design.md](./v2.8-migration-design.md) · Dry Run CTO 签字（2026-06-02）  
> **原则**：Rollback 优先于迁移脚本复杂度；不引入 Store Adapter 双栈

---

## 1. 备份（Phase C 第一步，强制）

在**任何**写入 `data/projects` / `data/studio.json` 之前：

```bash
BACKUP_ROOT="data/backup/phase-c-$(date +%Y%m%d-%H%M%S)"
mkdir -p "$BACKUP_ROOT"

cp data/studio.json "$BACKUP_ROOT/studio.json.bak"

for pid in data/projects/*/; do
  id=$(basename "$pid")
  tar -czf "$BACKUP_ROOT/projects-${id}.tar.gz" -C data/projects "$id"
done
```

| 产物 | 用途 |
|------|------|
| `studio.json.bak` | 恢复快照 / review / assets |
| `projects-{id}.tar.gz` | 恢复整项目（knowledge、verify、workspace） |

记录 `BACKUP_ROOT` 路径到迁移日志（`migration_report.json` 的 `backup_dir` 字段或终端输出）。

**规模参考（当前）**：`studio.json` ≈ 0.24 MB；2 项目 tar 合计 < 50 MB — 本地备份即可，无需外部存储。

## 2. 迁移后校验（validate）

Phase C 脚本结束前必须跑校验；**任一失败 → 进入 §3 回滚**，不继续 Cleanup。

| 检查项 | 通过标准 |
|--------|----------|
| **entities** | `knowledge/entities/characters.json` 存在；ACTIVE 条数 = Dry Run `active`；REJECTED 未写入 entities |
| **reject 一致性** | 磁盘上无 `我知` / `阳逸清没` / `个巨大的` 的 `char_*` id |
| **versions** | 每条 legacy snapshot 对应 `versions/{id}/files/` 且文件数 ≥ 原 `files[]`；`metadata.json` 无内嵌 content |
| **measurement** | `measurement/review/latest.json` 可读；`verify/verify_log.json` 保留 |
| **health_snapshot** | 已删除或不再被 reader 依赖（metrics rollup 替代） |
| **id 形态** | ACTIVE 实体 `id` 匹配 `char_[a-z0-9]{8}`，且 `id !== name` |

```bash
# 规划命令（Phase C 脚本内或独立）
node backend-node/scripts/migration-validate.mjs --project <id> --dry-run-report reports/<id>/
```

校验失败时退出码非 0，并打印失败项列表。

## 3. 回滚（restore）

**触发条件**：validate 失败、迁移脚本异常中断、或人工判定数据不可信。

```bash
BACKUP_ROOT="<上一步记录的目录>"

# 1. 恢复 studio 全局状态
cp "$BACKUP_ROOT/studio.json.bak" data/studio.json

# 2. 按项目恢复（覆盖迁移写入的目录）
for tgz in "$BACKUP_ROOT"/projects-*.tar.gz; do
  id=$(basename "$tgz" .tar.gz | sed 's/^projects-//')
  rm -rf "data/projects/$id"
  tar -xzf "$tgz" -C data/projects
done

# 3. 删除迁移新增路径（若已部分写入）
#    versions/、knowledge/entities/、measurement/ 等 — 由 tar 全量覆盖 projects/{id} 即可
```

**代码路径**：回滚后继续使用 **legacy readers**（读 `knowledge/*.json` 根目录、`studio.snapshots`），直至修复迁移脚本后重跑 Phase C。

| 不要做的事 | 原因 |
|------------|------|
| 半套 legacy + 半套 new 双写 | 无 Store Adapter；易不一致 |
| 在坏数据上跑 Cleanup（2C-4） | 会删掉仍可恢复的 legacy 写入 |

## 4. 与 Phase C / Cleanup 的关系

```text
node backend-node/scripts/migration-backup.mjs --all
    ↓
node backend-node/scripts/migration-phase-c.mjs --all --backup-dir <dir>
    ↓
node backend-node/scripts/migration-validate.mjs --all
    ↓
人工 Spot Check
    ↓
node backend-node/scripts/migration-enable-legacy-guard.mjs
    ↓
2C-4 Cleanup（删 legacy 写入路径）
```

**硬约束（不得合并脚本）**：

| # | 约束 |
|---|------|
| 1 | `migration-phase-c.mjs` 幂等：`migration_version: phase-c-v1` 已存在则退出 |
| 2 | `migration/id-mapping.json` 必须落盘 |
| 3 | `migration-validate.mjs` 含 referential integrity（relationship orphan → FAIL） |
| 4 | Cleanup 前执行 `migration-enable-legacy-guard.mjs` |

**Cleanup 前置条件**：validate 全绿 + 人工 spot-check `583e5628-24b` 创作中心可打开。
