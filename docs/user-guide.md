# 第一章：认识文匠 Studio

## 1.1 文匠 Studio 是什么？

**文匠 Studio** 是一款 **编剧级创作台**——面向网文作者、编剧与长篇叙事创作者的专业 AI 工作台。它的核心不是「替你写完」，而是帮你 **搭结构、改对白、迭代每一稿**，并在长篇连载中守住设定与伏笔的一致性。

产品由两层能力组成：

| 层次 | 名称 | 做什么 |
|------|------|--------|
| 创作层 | **写作工作台** | 三栏布局：文稿/大纲/设定 + 中央编辑器 + AI 对话 |
| 工程层 | **Story OS（故事工程）** | 右侧「写什么 → 写得好吗 → 怎么改」十条能力线，管理知识、质量、路线与改稿 |
| 引擎层 | **叙事引擎** | 基于 Claude Code CLI，读取你的工作区、Skill 与模型凭据，驱动对话、续写与审稿 |

想象一下，你有一个写作搭档：

- **记忆力超强**：角色、设定、伏笔都进知识库，随时可查
- **随时在线**：聊剧情、改对白、续写初稿
- **会挑毛病**：规则引擎 + Governor 决策，帮你找逻辑漏洞与前后矛盾
- **懂编剧**：剧本项目有场景看板、三幕结构、Fountain 导出与编剧室流水线

**核心理念：你负责创作，AI 负责辅助和把关。**

### 部署方式

文匠 Studio 可以：

- **本机运行**：浏览器打开 `http://localhost:8765`（默认端口）
- **Docker 部署**：在服务器上用容器运行，团队通过 `http://服务器IP:8765` 访问；数据保存在 Docker 数据卷中

无论哪种方式，**项目数据由你控制的服务端存储**，不会上传到第三方云端。

## 1.2 它能帮你做什么？

用一句大白话说：**你负责想故事，它帮你把故事写好、管好。**

具体来说：

| 你想做的事 | 文匠 Studio 怎么帮你 |
|-----------|---------------------|
| 我有个故事想法，想开始写 | 帮你建项目、搭大纲、设角色，然后你就可以开始写了 |
| 我写到一半写不下去了 | AI 读了你前面写的内容，帮你续写下面的情节 |
| 我觉得这段写得不好 | 选中那段文字，AI 帮你改写、扩写或润色 |
| 我忘了前面某个角色叫什么了 | 知识库里有所有角色的信息，随时查 |
| 我埋了一个伏笔，忘了在哪章了 | 伏笔看板帮你追踪每一条伏笔的状态 |
| 我担心故事后面和前面矛盾了 | AI 帮你做一致性检查，找出前后矛盾的地方 |
| 我想把小说导出成 Word 投稿 | 一键导出为 Word、EPUB、ZIP 等格式 |
| 我想写剧本而不是小说 | 切换到剧本模式，有专业的场景看板和编剧工具 |

## 1.3 开始之前，你需要准备什么？

| 需要的东西 | 说明 |
|-----------|------|
| 浏览器 | 推荐 Chrome 或 Edge |
| 文匠 Studio 已安装并运行 | 本机或 Docker，详见安装文档 |
| **Claude Code CLI** | 叙事引擎的运行依赖；未安装时对话与写稿不可用 |
| **模型凭据（API Key）** | 在 AI 中心配置，供 Claude Code 调用第三方模型（DeepSeek、MiMo、通义千问等） |
| 你的故事想法 | 哪怕只是一个模糊的念头也行 |

> **关于 AI 的说明：** 日常对话与写稿 **始终通过 Claude Code 叙事引擎** 执行。AI 中心「模型与连接」里配置的是 **模型凭据**（Base URL、协议、API Key），不是独立的备用聊天通道。第三章会详细说明配置方法与常见坑。

# 第二章：注册和登录

## 2.1 打开文匠 Studio

当文匠 Studio 运行起来后，打开你的浏览器（推荐用 Chrome 或 Edge），在地址栏输入：

```
http://localhost:8765
```

然后按回车键。

> **什么是 localhost:8765？** 这是文匠 Studio 在你电脑上运行的地址。`localhost` 意思是"我自己的电脑"，`8765` 是端口号（就像门牌号）。你不需要记这个，只要知道在浏览器里输入这个地址就能打开文匠 Studio 就行了。

按下回车后，你会看到一个**登录页面**。

## 2.2 注册一个新账号

如果你是第一次使用，你需要先注册一个账号。

**你会看到的界面：**

页面分成左右两半：
- **左边**：品牌介绍区，展示文匠 Studio 的特色功能
- **右边**：登录/注册卡片，上面有两个标签——"登录"和"注册账号"

**操作步骤：**

1. 看右边的卡片，点击 **"注册账号"** 这个标签
   - 点击后，卡片上的表单会切换成注册表单

2. 在 **"用户名"** 输入框里，输入你想用的用户名
   - 这是登录时用的，建议用英文字母和数字
   - 比如：`xiaoming`、`writer001`、`myname`
   - 不要用中文，不要用空格

3. 在 **"显示名称"** 输入框里，输入你想让别人看到的名字
   - 这个可以用中文
   - 比如：`小明`、`码字工`、`张三`
   - 这个名字会显示在界面上，比如"欢迎回来，小明"

4. 在 **"密码"** 输入框里，输入你的密码
   - 建议用一个你能记住但别人猜不到的密码

5. 在 **"确认密码"** 输入框里，再输入一遍同样的密码
   - 这是为了确认你没有打错密码

6. 检查一下所有信息都没问题，然后点击 **"注册"** 按钮

**如果注册成功：**
- 页面会自动跳转到**创作看板**（首页）
- 你会看到一个欢迎引导，介绍文匠 Studio 的主要功能

**如果注册失败：**
- 可能是用户名已经被别人用了，换一个试试
- 可能是两次密码输入不一致，重新检查一下
- 页面上会显示红色的错误提示，按照提示修改即可

## 2.3 下次怎么登录？

当你注册过一次之后，下次打开文匠 Studio，直接用你的账号登录就行：

1. 打开 `http://localhost:8765`
2. 确保在 **"登录"** 标签页
3. 输入你的**用户名**和**密码**
4. 点击 **"登录"** 按钮

登录成功后，你会进入创作看板。

## 2.4 退出登录

当你想退出时：
1. 看页面**右上角**，有一个显示你名字首字母的圆形头像
2. 点击这个头像
3. 会弹出一个下拉菜单
4. 点击 **"退出登录"**

退出后会回到登录页面。

## 2.5 Docker 与远程访问

如果你是通过 Docker 在服务器上部署文匠 Studio（例如团队共用一台云服务器）：

1. 在浏览器地址栏输入 `http://服务器IP:8765`（端口以实际配置为准）
2. 登录方式与本机相同：注册账号或使用管理员分配的账号
3. **数据持久化**：项目、对话、配置保存在 Docker 数据卷（如 `literarycraft-data`），重建容器不会丢数据，但删除数据卷会丢失一切
4. **升级镜像**：拉取或导入新镜像后，用 `docker compose up -d` 或项目提供的 `upgrade.sh` 重建容器；如有热修补丁，应以镜像版本为准

> **安全提示：** 暴露在公网的服务器请使用强密码，并考虑在反向代理后启用 HTTPS。

# 第三章：先别急着写——做好准备工作

在你开始写故事之前，我们先花几分钟配置一下 AI。这就像在写作业之前先把笔准备好一样。

## 3.1 理解叙事引擎与模型凭据

文匠 Studio 的 AI 能力由 **叙事引擎（Claude Code CLI）** 驱动，而不是浏览器直接调用大模型 API。

```
你输入消息 → WebSocket → 后端编排器 → 启动 claude CLI
                                      ↓
                            读取工作区 + Skill + 模型凭据
                                      ↓
                            流式返回续写 / 对话 / 改稿结果
```

因此需要同时满足两件事：

| 条件 | 在哪里看 | 不满足时的表现 |
|------|----------|----------------|
| Claude Code 已安装且可执行 | AI 中心 → **总览** → 「Claude Code 已连接」 | 对话面板显示「Claude Code 未连接」，无法聊天 |
| 模型凭据配置正确且已设为活跃 | AI 中心 → **模型与连接** → 测试连接成功 + 设为活跃 | 可能显示「模型连接测试失败」或 AI 身份/回复异常 |

**模型凭据** 告诉 Claude Code 去哪个服务商、用什么协议和 Key 推理。常见选择：

| 服务商 | 快捷模板 | 说明 |
|--------|----------|------|
| 小米 MiMo | MiMo OpenAI / MiMo Anthropic | 国内用户常用；**协议必须与 Base URL 匹配** |
| DeepSeek | DeepSeek | 性价比高 |
| OpenAI | OpenAI | GPT 系列 |
| 通义千问 | 通义千问 | 阿里云兼容接口 |

> **常见坑：** Base URL 填了 OpenAI 风格的 `/v1`，却选了 Anthropic 协议（或反过来），测试连接会失败或返回 HTML 404。务必使用模板一键填充，或手动保证 **协议 ↔ URL** 一致。

## 3.2 怎么配置模型凭据？

### 第一步：进入 AI 中心

1. 看页面**左侧**，有一排竖着的导航图标
2. 找到 **菱形图标（◈）**，悬停显示「AI 中心」，点击它

AI 中心有五个标签页：

| 标签 | 作用 |
|------|------|
| **总览** | 查看 Claude Code、活跃模型、默认 Skill 是否就绪 |
| **模型与连接** | 添加/测试/激活第三方模型凭据 |
| **本机技能** | 设置默认创作 Skill（叙事引擎的行为模板） |
| **发现安装** | 浏览并安装新的 Skill |
| **MCP 扩展** | 给 AI 挂载外部工具服务器 |

配置模型时，进入 **「模型与连接」**。

### 第二步：添加一个 AI 模型

1. 确保你在 **"模型与连接"** 标签页
2. 找到 **"添加模型"** 按钮，点击它
3. 会弹出一个表单

### 第三步：填写模型信息

你会看到一个表单，需要填写以下信息：

**① 快捷模板（推荐使用）**

表单顶部有几个快捷模板按钮：
- **OpenAI**：如果你用的是 ChatGPT/GPT-4
- **DeepSeek**：如果你用的是 DeepSeek（性价比高，推荐国内用户）
- **通义千问**：如果你用的是阿里云的通义千问
- **MiMo OpenAI**：小米的 MiMo 模型（OpenAI 协议）
- **MiMo Anthropic**：小米的 MiMo 模型（Anthropic 协议）

**建议：**

- 已有 **小米 MiMo** 账号：选 **MiMo Anthropic** 模板（与 Claude Code 原生协议更契合），或 **MiMo OpenAI**（兼容 OpenAI 接口）
- 其他国内用户：可选 **DeepSeek** 或 **通义千问**
- 已用 **CC Switch** 管理模型：在模型面板点击 **「从 CC Switch 导入」** 一键填入当前配置

点击模板后，大部分字段会自动填好，你只需要填 **API Key**。

**② 如果你没用模板，需要手动填写：**

| 字段 | 怎么填 | 举例 |
|------|--------|------|
| 显示名称 | 给这个模型起个你容易认的名字 | "我的 DeepSeek" |
| 协议类型 | 选"OpenAI 兼容"（大多数情况）或"Anthropic 兼容" | OpenAI 兼容 |
| API Key | 你的 AI 服务密钥（一串字母和数字） | `sk-abc123...` |
| Base URL | AI 服务的地址（含协议路径） | `https://api.deepseek.com/v1` |
| 模型 ID | 你要用的具体模型名称 | `deepseek-chat` |

> **API Key 是什么？** 就像你的 AI 服务的"密码"。你需要在 AI 服务商的网站上申请。比如 DeepSeek 的 API Key 在 https://platform.deepseek.com 申请。

### 第四步：测试连接

填好信息后，先别急着保存。点击表单里的 **"测试连接"** 按钮。

- 如果看到**绿色的「连接成功」**，说明凭据与协议正确
- 如果失败，按顺序检查：
  1. **协议类型** 是否与 Base URL 匹配（`/v1` → OpenAI；`/anthropic` → Anthropic）
  2. **API Key** 是否有效、未过期
  3. 网络能否访问该 Base URL（Docker 部署时注意容器出网）

### 第五步：保存并设为活跃

1. 测试成功后，点击 **「保存」**
2. 在模型列表中找到该模型，点击 **「设为活跃」**

「设为活跃」表示：此后 Claude Code 推理时使用这套凭据。

### 第六步：验证身份（换模型后必做）

更换活跃模型后，**建议新建一个对话会话**（会话选择器 → **+ 新建会话**），再问一句「你是什么模型？由谁提供？」。若 AI 仍自称 Claude 或旧模型，说明旧会话缓存了上下文——换新会话即可。

> **重要：** Claude Code 是 **唯一** 的执行通道；模型面板配置的是它使用的 **第三方推理后端**，不是「有 Claude Code 就不用配模型」。两者都需要：CLI 负责编排与读写工作区，模型凭据负责实际推理。

## 3.4 配置本机技能（叙事引擎）

Skill 是叙事引擎的「工作手册」——决定 AI 如何理解网文/剧本项目、何时调用规划与审查流程。

1. 进入 AI 中心 → **「本机技能」**
2. 查看已安装的 Skill 列表（如 literary-writer 等）
3. 在 **默认 Skill** 下拉框中选择日常创作用的技能，保存
4. 需要更多技能时，进入 **「发现安装」** 浏览并安装

配置后，Claude Code 会按默认 Skill 工作；仅在对话中明确说「改用 xxx」时才临时切换。

> **提示：** 首次使用建议在 **总览** 页确认三项绿灯：Claude Code 已连接、活跃模型已设置、默认 Skill 有效。

## 3.3 认识左侧导航栏

配置完模型与 Skill 后，熟悉一下全局导航结构。

**左侧导航栏**是你在整个软件中移动的主要方式。它是一排竖着的图标，从上到下依次是：

| 图标 | 名称 | 点击后去哪里 | 干什么用的 |
|------|------|-------------|-----------|
| ◉ | 创作看板 | 首页仪表盘 | 看你的写作统计数据 |
| ◫ | 项目库 | 项目列表 | 管理你所有的写作项目 |
| ◑ | 审稿中心 | 审稿页面 | 对作品进行深度质量审查 |
| ◧ | 素材中心 | 素材页面 | 管理跨项目的素材（角色卡、地点等） |
| ⧉ | 项目版本 | 版本页面 | 备份和恢复项目版本 |
| ◈ | AI 中心 | AI 管理页面 | 配置 AI 模型、技能、扩展 |
| 💬 | 留言板 | 留言页面 | 和其他用户交流 |

> **现在你不需要点每一个**，知道它们在哪就行。后面用到的时候会详细教你。

# 第四章：创建你的第一个项目

## 4.1 什么是"项目"？

在文匠 Studio 里，**项目 = 一部作品**。

比如：
- 你要写一本叫《斗破苍穹》的小说 → 这是一个项目
- 你要写一个叫《流浪地球》的剧本 → 这是一个项目
- 你要写一个短视频脚本系列 → 这也是一个项目

每个项目是完全独立的，互不影响。

## 4.2 怎么创建项目？

### 第一步：进入项目库

1. 点击左侧导航栏的 **"项目库"**（图标 ◫）
2. 你会进入项目库页面

**你会看到的界面：**

- **顶部**：一个大横幅，写着文匠 Studio 的品牌介绍
- **工具栏**：有搜索框、筛选按钮、排序按钮、视图切换按钮
- **项目列表区域**：现在应该是空的（因为你还没有项目）

### 第二步：点击"新建项目"

在页面的右上角（或工具栏区域），找到 **"新建项目"** 按钮，点击它。

会弹出一个**创建项目模态框**（一个弹出窗口）。

### 第三步：填写项目信息

模态框里有好几个字段，我一个一个解释：

**① 项目标题**

就是你的作品名字。比如：
- 《重生之都市修仙》
- 《三体》
- 《我的短视频脚本》

直接在输入框里打字就行。

**② 作品类型（非常重要！）**

这是一个下拉框，你需要选择你的作品类型。这个选择**非常重要**，因为它决定了你后面用什么样的编辑器和工具。

| 选项 | 选它的场景 | 你会得到什么 |
|------|-----------|-------------|
| 长篇小说 | 网文连载、长篇出版物 | 普通的文字编辑器 + AI 辅助 |
| 短篇小说 | 短篇故事、微型小说 | 同上，适合较短的作品 |
| 电影剧本 | 院线电影、网络电影 | 专业编剧编辑器（场景看板、三幕结构） |
| 剧集剧本 | 电视剧、网剧、动画 | 专业编剧编辑器（分集管理、伏笔追踪） |
| 短视频脚本 | 抖音、B站短视频 | 分镜编辑器（镜头管理、节奏分析） |

> **怎么选？** 如果你写的是小说（网文、出版物），选"长篇小说"或"短篇小说"。如果你写的是剧本，根据是电影还是剧集来选。如果你做短视频，选"短视频脚本"。
>
> **不确定怎么办？** 选"长篇小说"，这是最通用的选项。后面随时可以改。

**③ 创作模式**

| 选项 | 选它的场景 |
|------|-----------|
| 从零开始 | 你有一个全新的想法，从头开始写 |
| 改写续写 | 你已经有一些稿子了，想改写或继续往下写 |
| 导入已有 | 你有现成的 Word 文档或文本文件，想导入进来 |

> **第一次用建议选「从零开始」。** 如果你已经有稿子了，选「改写续写」或「导入已有」，后面会教你如何导入。改写续写模式下，工作台左侧会多出 **试验稿** 与 **旧稿** 目录。

**④ 题材/风格（可选）**

比如：玄幻、都市、科幻、言情、悬疑、历史……

这个字段是可选的，不填也没关系。但填了之后，AI 能更好地理解你的故事类型，给出更合适的建议。

**⑤ 简介（可选）**

用一两句话描述你的故事。比如：
- "一个普通人穿越到修仙世界，从零开始修炼的故事"
- "2077年，人类发现了一颗宜居星球，但第一批移民遭遇了未知生物"

这个也是可选的，不填也没关系。

### 第四步：点击创建

检查一下信息没问题后，点击模态框底部的 **"创建"** 按钮。

创建成功后：
- 模态框会关闭
- 你会在项目列表中看到你刚创建的项目
- 页面可能会自动跳转到项目的工作台

**恭喜你！你已经创建了你的第一个项目！** 🎉

## 4.3 项目卡片的操作菜单

在项目库中，每个项目卡片右上角有 **⋯** 按钮（不是右键菜单）。点击后可：

| 菜单项 | 作用 |
|--------|------|
| 置顶 / 取消置顶 | 把常用项目固定在列表顶部 |
| 共享权限 | 邀请其他用户，设置只读或可编辑（需管理权限） |
| 重命名 | 修改项目标题 |
| 编辑简述 | 修改项目简介 |
| 导入续写 | 导入 docx 等文稿继续写 |
| 状态 | 写作中 / 暂停 / 完稿等 |
| 归档 / 移出归档 | 收纳已完成项目 |
| 删除 | 不可恢复，需输入标题确认 |

# 第五章：认识你的写作工作台

## 5.1 工作台是什么？

**写作工作台**是你以后写故事的主要场所。你可以把它想象成你的"书桌"——左边放参考资料，中间是稿纸，右边坐着你的 AI 搭档。

## 5.2 工作台的布局

当你进入一个项目后，你会看到一个**三栏布局**：

```
┌──────────┬──────────────────────────┬────────────────────┬──────────────┐
│ 左侧菜单  │       中央编辑区           │   右侧 AI 对话面板    │ Story OS 导航 │
│          │                          │                    │ （最右侧）    │
│ ☰ 文稿   │   [工具栏]                │   [会话选择器]       │ 写什么        │
│ ◇ 大纲   │   [编辑器 / 编剧界面]      │   [连接状态]         │ 写得好吗      │
│ ◎ 设定   │                          │   [消息列表]         │ 怎么改        │
│ ↓ 导出   │                          │   [输入框]           │              │
└──────────┴──────────────────────────┴────────────────────┴──────────────┘
```

让我一个一个介绍每个区域。

## 5.3 左侧菜单

左侧菜单是一排竖着的图标按钮，从上到下依次是：

| 图标 | 名称 | 作用 |
|------|------|------|
| ☰ | 文稿 | 正文章节列表；工具栏有 **+ 新建** |
| ◇ | 大纲 | 大纲文件列表；通过顶部 **更多 → 导入** 添加 |
| ◎ | 设定 | 设定文件列表；通过 **更多 → 导入** 添加 |
| ✎ | 试验稿 | 仅「改写续写」模式：存放试验性修改 |
| ⊟ | 旧稿 | 仅「改写续写」模式：归档旧版本 |
| ↓ | 导出 | 打开导出模态框 |

> **先有文稿再管大纲/设定：** 若项目还没有任何章节，大纲和设定入口会呈灰色不可用，避免空项目里堆设定却无处落笔。

## 5.4 中央编辑区

这是你写文字的地方，占了屏幕最大的面积。

**顶部工具栏包含：**

| 元素 | 作用 |
|------|------|
| ← 返回链接 | 点击回到项目库 |
| 项目标题 | 显示当前项目的名称 |
| 设置按钮 ⚙️ | 修改项目信息（标题、类型等） |
| 上传按钮 📤 | 导入文档到项目中 |
| 快捷键按钮 ⌨️ | 查看键盘快捷键 |
| 诊断按钮 🔍 | 查看项目诊断报告 |
| 导出按钮 📥 | 导出作品 |

**编辑器本身：**

- 就是一个大的**文本输入区域**
- 你可以在里面打字、写故事
- 支持 Markdown 格式（一种简单的文本格式，用 `#` 表示标题，用 `**` 表示加粗等）
- 有两种模式可以切换：**编辑模式**（直接打字）和**预览模式**（看渲染效果）

**底部信息栏：**
- 显示当前文件的**字数**
- 显示**自动保存状态**（比如"已保存"或"保存中..."）

## 5.5 右侧 AI 对话面板

这是你和 AI 搭档聊天的地方。

**从上到下依次是：**

**① 会话选择器（最顶部的下拉框）**

- 用来切换不同的对话
- 你可以为每一章创建一个独立的对话
- 也可以有一个全局对话，讨论整体故事走向
- 点击下拉框可以切换、创建或删除会话

**② 连接状态栏**

- 显示 AI 是否在线
- **绿色圆点 + "已连接"** = 一切正常，可以聊天
- **黄色圆点 + "连接中..."** = 正在连接，等一下
- **红色圆点 + "连接失败"** = 有问题，点击重连按钮

**③ 消息历史（中间的大区域）**

- 显示你和 AI 的对话记录
- 你的消息在右边，AI 的消息在左边
- AI 回复时是**流式显示**的——你会看到它一个字一个字地"打"出来

**④ 快捷建议**

- 输入框上方有 **「快捷建议 ▼」** 折叠区
- 建议内容按 **作品类型** 和 **创作模式** 动态生成（如「梳理人物关系」「对照大纲检查偏离」）
- 点击一条建议即发送，无需自己打字

**⑤ 消息输入框（最底部）**

- 在这里输入你想问 AI 的问题
- 按 **Enter** 发送
- 按 **Shift + Enter** 换行（不发送）

## 5.6 如果是剧本项目，编辑器会不同

如果你创建项目时选的是"电影剧本"、"剧集剧本"或"短视频脚本"，中央编辑区会变成一个**专业编剧界面**：

- **电影剧本**：你会看到一个"场景看板"，把场景分成三幕（建置、对抗、解决）
- **剧集剧本**：你会看到"剧集列表"、"伏笔追踪"、"角色弧线"三个标签页
- **短视频脚本**：你会看到一个"分镜看板"，管理每个镜头

这些是编剧专用的功能，后面在第十四章会详细讲。

## 5.7 认识 Story OS（故事工程）

进入项目后，屏幕**最右侧**有一条竖向导航，顶部 **「写作」** 按钮回到三栏工作台，下方是 Story OS——按创作场景分成三组：

### 写什么（今日任务与路线）

| 入口 | 作用 |
|------|------|
| **今日建议** | 今日任务、诊断建议；需先 **快速同步** 才有数据 |
| **创作路线** | 故事目标 + 章节路线图；可 AI 生成后续 N 章规划 |

### 写得好吗（质量、节奏与知识）

| 入口 | 作用 |
|------|------|
| **作品质量** | 叙事引擎审稿、规则/语义审查、章节启发式评分 |
| **悬念分析** | 每章悬念强度、张力曲线、线索分类 |
| **节拍大纲** | 把一章拆成多个场景节拍，支持一键从节拍写作 |
| **作品知识** | 角色卡片、伏笔看板、知识查询、**从设定集同步** |

### 怎么改（计划、角色与编剧室）

| 入口 | 作用 |
|------|------|
| **修改计划** | 待确认 / 执行中 / 已完成的改稿任务（有未处理时会显示角标） |
| **角色工坊** | 心理模型、声纹 DNA、关系图 |
| **设定圣经** | 结构化世界观表单 |
| **编剧室** | Canon 规则、叙事记忆、Creator→Critic→Governor 流水线 |

**推荐工作流：**

1. 工作台写好大纲与设定 → **作品知识 → 从设定集同步**
2. **今日建议 → 快速同步** → 看今日任务
3. 回到工作台写作 → 阶段性去 **作品质量** 或 **审稿中心** 深度检查
4. 审查问题 → 转为 **修改计划** → 回工作台执行

# 第六章：在动笔之前——搭建故事骨架

> **重要提示：** 这一章的内容不是必须的，但强烈建议你做。就像盖房子要先打地基一样，先把故事的基本框架搭好，后面写起来会顺畅很多。
>
> 如果你迫不及待想开始写，可以先跳到第七章，等写了几章之后再回来补这些。

## 6.1 第一步：写一个大纲

### 什么是大纲？

大纲就是你的故事的"骨架"。它不写具体的对话和描写，只写每一章大概发生什么。

**打个比方：** 如果你的故事是一栋楼，大纲就是建筑图纸。你不需要在图纸上画每一面墙的花纹，但你需要知道这栋楼有几层、每层是什么功能。

### 怎么写大纲？

1. 在工作台中，点击左侧菜单的 **「大纲」**（◇）
2. 若列表为空，按提示通过顶部 **「更多 → 导入」** 导入已有大纲，或先在工作台创建至少一章文稿后再添加
3. 导入或创建 `.md` 大纲文件后，在中央编辑器中编写
4. 大纲支持 **结构树**：可拖拽调序、点击节点编辑梗概（回写 Markdown）

### 大纲应该写什么？

不需要写得很详细，但建议包含以下几个部分：

```markdown
# 故事大纲

## 故事简介
用两三句话概括你的故事讲了什么。

## 主要角色
- 主角：名字、身份、目标
- 反派：名字、身份、目的
- 重要配角：名字、和主角的关系

## 故事走向
### 开头
故事怎么开始？主角在什么处境下出场？

### 发展
中间发生了什么关键事件？

### 高潮
最紧张、最精彩的部分是什么？

### 结局
故事怎么结束？

## 核心冲突
故事的主要矛盾是什么？主角最大的困难是什么？
```

> **写大纲的小技巧：**
> - 不要追求完美，先写一个大概的框架，后面可以随时改
> - 不需要写得很长，几百字就够
> - 如果你实在不知道怎么写，可以先和 AI 聊聊，让它帮你出主意

### 和 AI 一起写大纲

如果你不知道怎么写大纲，可以找 AI 帮忙：

1. 点击右侧的 AI 对话面板
2. 在输入框里输入：**"我想写一个玄幻小说，帮我设计一个故事大纲"**（替换成你自己的题材）
3. AI 会给你一个大纲框架
4. 你可以在它的基础上修改，变成你自己的故事

## 6.2 第二步：建立设定集

### 什么是设定集？

设定集就是你的故事的"百科全书"。它记录了你故事中所有的"事实"——角色信息、世界观设定、力量体系等。

### 为什么需要设定集？

因为 AI 在帮你写作时，需要知道这些"事实"才能保持一致性。

**举个例子：**
- 你在设定集里写了"主角怕水"
- 后面你写到第 20 章，AI 帮你续写
- 如果没有设定集，AI 可能会让主角开心地去游泳
- 有了设定集，AI 会记得"主角怕水"，不会犯这种错误

### 怎么建立设定集？

1. 在工作台中，点击左侧菜单的 **「设定」**（◎）
2. 通过 **「更多 → 导入」** 添加设定文件，或从空列表提示导入 docx/md
3. 建议逐步创建以下文件（不必一次全建）：

**建议创建的文件：**

| 文件名 | 写什么内容 |
|--------|-----------|
| 角色设定.md | 每个角色的详细信息（见下面的模板） |
| 世界观设定.md | 故事发生的世界是什么样的 |
| 力量体系.md | 如果有超能力/魔法/武力系统，在这里描述 |
| 势力关系.md | 各个组织、家族、国家之间的关系 |

### 角色设定模板

你可以在"角色设定.md"里这样写：

```markdown
# 角色设定

## 主角：林风
- 年龄：18岁
- 外貌：黑发，中等身材，左眼下方有一道小疤
- 性格：外表冷漠，内心善良，对朋友极其忠诚
- 背景：孤儿，从小在街头长大，被一位老道士收养
- 能力：天生灵根，擅长火系法术
- 目标：找到失散的妹妹
- 弱点：过于重感情，容易被利用
- 口头禅："无所谓"

## 反派：赵天明
- 年龄：40岁
- 外貌：中年男子，总是穿着黑色长袍
- 性格：阴险狡诈，野心勃勃
- 背景：曾经是修仙界的天才，因为修炼邪术被逐出师门
- 能力：精通黑暗法术，实力深不可测
- 目标：收集九件神器，打开通往魔界的大门
```

> **设定集不求全，但求有。** 你不需要一开始就写得很完美。先把主要角色和基本设定写上，后面边写边补充。

## 6.3 第三步：了解"设定圣经"

除了上面那种自由格式的设定文件，文匠 Studio 还提供了一个**结构化的世界观管理工具**，叫做"设定圣经"。

### 设定圣经和设定文件有什么区别？

| 设定文件 | 设定圣经 |
|----------|----------|
| 自由格式的 Markdown 文档 | 结构化的表单和分区 |
| 适合写详细的角色描述、世界观说明 | 适合快速填写和查阅关键信息 |
| AI 读取时需要从文本中提取信息 | AI 可以直接读取结构化数据 |

### 怎么使用设定圣经？

1. 在工作台中，点击**右侧导航栏**的 **"设定圣经"**
2. 你会看到一个表单，包含以下字段：

**核心元数据（顶部）：**

| 字段 | 填什么 | 举例 |
|------|--------|------|
| 标题 | 你的作品名 | 《重生之都市修仙》 |
| 题材 | 故事类型 | 玄幻 |
| Logline | 一句话概括故事 | "一个普通大学生意外获得修仙传承，在都市中修炼成长的故事" |
| 背景设定 | 故事发生的时间地点 | "现代都市，2024年的上海" |
| 基调 | 故事的感觉 | 热血、轻松、黑暗、温暖 |
| 主题 | 故事想表达什么 | "成长、友情、正义" |

**世界观分区（下方）：**

你可以添加多个分区，每个分区选择一个类型：

| 类型 | 适合写什么 |
|------|-----------|
| 世界观 | 整个世界的基本设定 |
| 魔法体系 | 修仙/魔法/超能力的规则 |
| 政治 | 国家、势力的政治格局 |
| 历史 | 世界的历史背景 |
| 风俗 | 世界的风俗习惯 |

每个分区有**标题**和**内容**两个字段。比如：

- 类型：魔法体系
- 标题：修仙等级
- 内容：炼气期 → 筑基期 → 金丹期 → 元婴期 → 化神期 → 大乘期 → 渡劫期

## 6.4 第四步：同步知识库

当你写好了大纲和设定后，需要让文匠 Studio 的"知识库"把它们读一遍，建立索引。

### 什么是知识库？

知识库是文匠 Studio 的"大脑"。它会从你的大纲和设定文件中提取关键信息（角色、地点、伏笔等），建立一个结构化的数据库。这样 AI 在帮你写作时，就能快速查找和引用这些信息。

### 怎么同步？

1. 在工作台中，点击**右侧导航栏**的 **"作品知识"**
2. 你会看到几个区域：角色卡片、伏笔看板、查询界面等
3. 点击 **「从设定集同步」** 按钮
4. 系统会扫描大纲与设定文件，提取角色、伏笔等结构化信息
5. 同步完成后，在「角色卡片」区域查看识别结果

### 同步后你会看到什么？

**① 角色卡片**
- 每个被识别出来的角色会有一张卡片
- 卡片上显示角色的名字、角色定位、备注
- 你可以直接在这里编辑

**② 伏笔看板**
- 三列：已埋设 / 待解决 / 已回收
- 现在应该是空的（因为你还没开始写，还没埋伏笔）
- 等你写了几章之后，伏笔会逐渐出现

**③ 查询界面**
- 一个搜索框，你可以输入关键词查询知识库
- 比如输入"林风"，会显示所有和林风相关的信息

> **什么时候需要重新同步？** 每次你大幅修改了设定文件后，建议重新同步一次。这样知识库才能反映最新的设定。

# 第七章：开始写第一章

好了，准备工作做得差不多了。现在开始真正的写作！

## 7.1 创建你的第一个章节

1. 确保你在工作台中
2. 点击左侧菜单的 **"文稿"** 图标
3. 中央编辑区会显示文稿文件列表
4. 若是新项目，中央区显示 **空状态引导**，可「导入 docx」或「新建一章」
5. 在文稿侧栏点击 **「+ 新建」**，或按引导按钮
6. 在弹窗 **「新建文稿」** 中输入章节标题（如「第一章 初入江湖」）
7. 确认后即可在编辑器中写作

创建成功后，中央编辑区会变成一个大的文本编辑区域，你可以在里面开始写故事了。

## 7.2 开始打字

现在你可以直接在编辑器里打字写你的故事了。

### 编辑器的基本操作：

| 你想做的事 | 怎么做 |
|-----------|--------|
| 输入文字 | 直接打字 |
| 换行 | 按 Enter 键 |
| 加粗文字 | 用 `**` 包裹，比如 `**这是加粗的文字**` |
| 变成标题 | 在文字前面加 `#`，比如 `# 大标题` 或 `### 小标题` |
| 保存 | 自动保存（停止打字 2.5 秒后自动存），也可以按 Cmd/Ctrl + S |
| 撤销 | 按 Cmd/Ctrl + Z |
| 切换到预览模式 | 点击编辑器上方的"预览"按钮 |
| 切换回编辑模式 | 点击"编辑"按钮 |

### 关于 Markdown

文匠 Studio 使用 **Markdown** 格式。Markdown 是一种简单的文本格式，用一些符号来表示格式。

**你不需要精通 Markdown！** 你完全可以像在记事本里一样，直接打字写故事。Markdown 格式是可选的，如果你不想用，完全可以不用。

如果你想了解基本的 Markdown：

| 你输入的 | 显示效果 |
|----------|----------|
| `# 标题` | 大标题 |
| `## 二级标题` | 中等标题 |
| `**加粗**` | **加粗** |
| `*斜体*` | *斜体* |
| `这是一段普通文字` | 这是一段普通文字 |

### 调整字号

如果觉得编辑器里的字太小或太大：
1. 找到编辑器上方的**字号按钮**
2. 点击它，会显示几个选项：15px、16px、18px、20px
3. 选择你觉得舒服的大小
4. 这个设置会自动记住，下次打开还是这个大小

## 7.3 切换章节

当你写完了第一章，想写第二章时：

**方法一：** 在文稿侧栏点击 **「+ 新建」**，创建下一章

**方法二：** 用键盘快捷键 **Cmd/Ctrl + →**（右箭头）切换到下一章

**方法三：** 在左侧文件列表中直接点击你想编辑的章节

## 7.4 专注模式

如果你觉得左右两边的面板太分散注意力，可以开启**专注模式**：

1. 按键盘上的 **F** 键
2. 左侧菜单和右侧 AI 面板会隐藏，只留下中央编辑区
3. 再按一次 **F** 键，恢复原来的布局

专注模式下，你可以更专心地写作，不受其他界面元素的干扰。

# 第八章：让 AI 帮你写作

写了几段之后，你可能会遇到这些情况：
- "我写不下去了，下面该怎么发展？"
- "这段对话太平淡了，怎么改得更有张力？"
- "我想让 AI 帮我写一版初稿"

这时候就轮到 AI 出场了。

## 8.1 和 AI 对话

最简单的使用方式就是**直接和 AI 聊天**。

### 怎么做？

1. 看右侧的 AI 对话面板
2. 确保连接状态是**绿色的"已连接"**
3. 在底部的输入框里，输入你想问的问题
4. 按 **Enter** 发送

### 你可以问 AI 什么？

**关于剧情：**
- "这一章接下来该怎么写？"
- "我觉得这个冲突不够强，怎么改？"
- "主角在这个场景下应该有什么反应？"
- "帮我设计一个反转情节"

**关于角色：**
- "帮我设计一个反派角色"
- "这个角色的性格应该怎么发展？"
- "这两个角色之间的关系应该怎么变化？"

**关于写作技巧：**
- "这段打斗场面怎么写得更精彩？"
- "怎么写一段感人的对话？"
- "这段描写太啰嗦了，怎么精简？"

**关于整体故事：**
- "我的故事节奏是不是太快了？"
- "帮我分析一下目前的故事线"
- "这个伏笔应该怎么回收？"

### AI 回复的方式

当你发送消息后，AI 会开始回复。你会看到：
- AI 的回复是**一个字一个字地出现**的（流式输出）
- 在 AI 还在回复时，你可以随时点击 **"停止"** 按钮中断它
- 回复完成后，消息会保留在对话历史中

### 使用快捷建议

展开输入框上方的 **「快捷建议 ▼」**。建议因项目类型而异，例如：

- 小说：**「根据最新文稿，讨论接下来最合理的续写方向」**
- 剧本：**「这一场戏的功能是什么？推进/揭示/转折？」**
- 改写模式：**「指出当前稿的主要结构问题，并给 2 个重写方向」**

也可直接在输入框说「帮我续写这一章」「续写第一章」等自然语言指令。

## 8.2 让 AI 续写

这是最常用的功能之一。当你写到某个地方写不下去时，可以让 AI 帮你续写。

### 怎么做？

**方法一：直接告诉 AI**

在对话框里输入类似这样的话：
- "帮我续写这一章"
- "从这里继续写下去"
- "接着写 500 字"

**方法二：使用快捷建议**

在 **「快捷建议」** 中选一条与续写相关的建议，或直接输入「帮我续写这一章」。

### AI 续写的原理

你可能会好奇：AI 怎么知道该写什么？

AI 在续写时，会读取以下信息：
1. **你的大纲**：知道故事应该往什么方向发展
2. **你的设定集**：知道角色信息和世界观
3. **知识库**：知道前面发生了什么
4. **当前章节**：知道你写到哪了
5. **对话历史**：知道你们之前讨论过什么

然后它会根据这些信息，生成最合理的续写内容。

### 续写完成后

AI 续写完成后，你会看到一个**写作结果**。这时候你有三个选择：

| 选择 | 操作 | 适合场景 |
|------|------|----------|
| 打开为新文件 | AI 写的内容保存为一个新章节 | 你想保留原稿，把 AI 的版本单独存放 |
| 替换当前内容 | 用 AI 的内容替换你正在编辑的章节 | 你觉得 AI 写得更好，想直接用 |
| 丢弃 | 不采用 AI 的内容 | 你不满意，想自己写 |

> **建议：** 如果你不确定 AI 写得好不好，先选"打开为新文件"。这样你可以对比两个版本，再决定用哪个。

## 8.3 和 AI 讨论你的故事

除了让 AI 帮你写，你还可以和它**讨论**你的故事。

### 举个例子：

你：「你觉得主角在这个场景下的反应合理吗？他刚失去了最好的朋友，但下一章就开心地去冒险了，是不是太突兀了？」

AI：「确实有些突兀。失去挚友后，角色应该有一段悲伤和愤怒的过程。我建议在下一章开头加一段内心独白，展示他的悲痛，然后通过某个触发事件（比如发现朋友留下的遗物）让他重新振作起来……」

这种讨论对打磨故事非常有帮助。AI 就像一个永远有空的编辑，你随时可以找它聊。

## 8.4 会话管理

### 什么是会话？

会话就是你和 AI 的一次"对话"。你可以创建多个会话，每个会话有不同的上下文。

### 为什么要用多个会话？

- **按章节分**：每一章一个会话，这样 AI 的注意力集中在当前章节
- **按话题分**：一个会话讨论剧情，一个会话讨论角色，一个会话讨论文笔
- **全局会话**：一个会话讨论整体故事走向

### 怎么管理会话？

1. 看 AI 对话面板顶部的**会话选择器**（下拉框）
2. 点击它，你会看到所有已创建的会话
3. 你可以：
   - **切换会话**：点击一个会话名称
   - **创建新会话**：点击 **「+ 新建会话」**
   - **删除会话**：在会话列表中删除

> **换模型后请新建会话**，避免旧对话仍绑定上一套模型身份。

# 第九章：修改和润色你写的内容

写完一段之后，你可能觉得某些地方写得不好。不需要手动重写——可以让 AI 帮你改。

## 9.1 选中文本进行 AI 编辑

### 怎么做？

1. 在编辑器中，用鼠标**选中一段文字**
   - 按住鼠标左键拖动，选中你想修改的文字
   - **注意：** 至少要选中 8 个字符（大约 4 个汉字）

2. 选中后，你会看到一个**浮动操作栏**出现在选中文字的上方

3. 操作栏上有几个按钮：

| 按钮 | 做什么 | 什么时候用 |
|------|--------|-----------|
| AI 改写 | AI 用不同的方式重写这段话 | 觉得这段写得不好，但不知道怎么改 |
| 扩写 | AI 把这段话写得更详细 | 觉得这段太简略了，想展开 |
| 润色 | AI 优化这段话的文笔 | 觉得这段太平淡了，想让它更精彩 |
| 与 AI 讨论 | 把这段话发给 AI 讨论 | 想听听 AI 对这段的看法 |

4. 点击你需要的操作

### 等待 AI 处理

点击后，AI 会开始处理你的请求。你会看到：
- 一个加载提示，告诉你 AI 正在工作
- 处理完成后，会弹出一个**对比预览窗口**

### 对比预览窗口

这个窗口会并排显示：
- **左边**：你原来的文字
- **右边**：AI 修改后的文字

你可以仔细对比，看看 AI 改了什么。

### 接受或拒绝

- **如果满意**：点击"接受"或"应用"按钮，AI 的修改会替换你原来的文字
- **如果不满意**：点击"取消"或"关闭"，你的文字保持不变

> **小贴士：** AI 不是每次都改得好。如果觉得改得不如原来，大胆拒绝就行。你才是故事的主人，AI 只是助手。

## 9.2 什么时候该用 AI 编辑？

| 场景 | 推荐操作 |
|------|----------|
| 这段对话太生硬了 | 选中对话 → AI 改写 |
| 这段描写太平淡了 | 选中描写 → 润色 |
| 这段情节太简略了 | 选中情节 → 扩写 |
| 这段打斗不够精彩 | 选中打斗 → 润色 |
| 这段转折太突兀了 | 选中转折段落 → 与 AI 讨论 |

# 第十章：管理你的故事世界

随着你写的内容越来越多，你需要开始"管理"你的故事——追踪角色发展、检查伏笔、维护设定。

## 10.1 作品知识页面

### 什么是作品知识？

作品知识是你的故事的"数据库"。它记录了所有角色、关系、时间线、伏笔、地点等信息。

### 怎么打开？

点击右侧导航栏的 **"作品知识"**

### 你会看到什么？

**① 角色卡片区域**

这里显示你故事中所有被识别出来的角色。每个角色一张卡片：
- 显示角色的**名字**和**角色定位**（主角、反派、配角等）
- 显示**备注**信息

**怎么添加新角色？**
1. 点击 **"添加"** 按钮
2. 输入角色名字
3. 选择角色定位
4. 填写备注（可选）
5. 点击保存

**怎么编辑已有角色？**
1. 点击角色卡片上的 **"编辑"** 按钮
2. 修改信息
3. 点击保存

**② 伏笔看板**

这是一个**看板式**的界面，分成三列：

| 列名 | 含义 |
|------|------|
| 已埋设 | 你已经在故事中暗示了，但还没有揭示的伏笔 |
| 待解决 | 需要在后面的章节中回收的伏笔 |
| 已回收 | 已经在故事中揭示了的伏笔 |

**怎么添加伏笔？**
1. 点击看板顶部的 **"添加伏笔"** 按钮
2. 输入伏笔的描述
3. 选择初始状态（通常是"已埋设"）
4. 保存

**怎么移动伏笔？**
- 直接**拖拽**伏笔卡片，从一列拖到另一列
- 比如：当你在故事中揭示了某个伏笔，就把它从"已埋设"拖到"已回收"

> **为什么要管理伏笔？** 伏笔是好故事的灵魂。你埋了一个伏笔但忘了回收，读者会失望。伏笔看板让你一目了然地看到哪些伏笔还需要处理。

**③ 查询界面**

页面底部有一个搜索框。你可以：
1. 输入关键词（比如角色名字）
2. 选择查询类型（通用、角色、关系、时间线、伏笔、地点）
3. 点击搜索
4. 查看搜索结果

**④ 「从设定集同步」按钮**

修改设定文件后，点击此按钮让知识库重新读取并更新索引。

## 10.2 角色工坊

### 什么是角色工坊？

角色工坊是一个**深度角色开发工具**。它不只是记录角色的基本信息，还帮你从心理层面塑造角色。

### 怎么打开？

点击右侧导航栏的 **"角色工坊"**

### 角色工坊能做什么？

**① 心理模型**

每个角色有 5 个心理维度，帮助你理解角色的内心世界：

| 维度 | 含义 | 举例 |
|------|------|------|
| Want（想要） | 角色表面追求的目标 | "我要成为最强的修仙者" |
| Need（需要） | 角色真正需要的成长（通常他自己不知道） | "我需要学会信任他人" |
| Ghost（心魔） | 过去的创伤记忆 | "小时候亲眼看着家人被杀" |
| Wound（创伤） | 心理上的伤疤 | "无法信任任何人" |
| Lie（谎言） | 角色相信的错误信念 | "只有变强才能保护身边的人" |

> **为什么这些很重要？** 因为好的角色不是完美的人，而是有缺陷、有成长的人。这五个维度帮你设计出有深度的角色。

**② 声纹DNA**

声纹DNA 是 AI 从你的手稿中分析出的角色说话方式。

**怎么训练？**
1. 在角色工坊中选择一个角色
2. 点击 **"训练"** 按钮
3. AI 会读取你手稿中这个角色的所有对话
4. 分析完成后，你会看到这个角色的说话特点：
   - 语调（严肃/幽默/冷淡）
   - 正式度
   - 口头禅
   - 禁用词（这个角色绝不会说的话）
   - 平均句长
   - 疑问句比例

**声纹DNA 有什么用？**

以后 AI 帮你写这个角色的对话时，会按照这个角色的语言风格来写。这样每个角色说话都有自己的特点，不会千人一面。

**③ 角色关系图**

一个可视化的图，展示所有角色之间的关系。像一张蜘蛛网，每个角色是一个节点，关系是连线。

## 10.3 设定圣经

前面在第六章已经介绍过了。简单来说，设定圣经是一个结构化的世界观管理工具，帮你快速填写和查阅故事的关键设定。

# 第十一章：检查你的作品质量

写了一段时间后，你可能想知道："我的故事写得怎么样？有没有什么问题？"

文匠 Studio 提供了几个工具帮你检查作品质量。

## 11.1 作品质量页面

### 怎么打开？

点击右侧导航栏的 **"作品质量"**

### 你会看到什么？

作品质量页整合了 **叙事引擎审稿** 与 **章节启发式评分**，注意两套维度名称不同：

**① 叙事引擎 6 维度（EngineCritic / 审稿嵌入区）**

| 维度 | 检查什么 |
|------|----------|
| 结构 | 故事结构是否完整 |
| 角色 | 角色塑造是否一致 |
| 场景 | 场景描写与转换 |
| 叙事压力 | 悬念与张力 |
| 声音/对白 | 角色语言是否有个性 |
| 连续性/主题 | 前后矛盾与主题表达 |

每项标记为 **HARD**（严重）/ **SOFT**（建议）/ **PASS**（通过），并有 **Governor 决策**：通过 / 建议修订 / 建议重写。

**② 近期章节评分（启发式，非 LLM）**

对每章另有一套 6 维度打分，用于快速浏览：

| 维度 | 含义 |
|------|------|
| 人物塑造 | 角色刻画 |
| 剧情推进 | 情节发展 |
| 节奏控制 | 段落与节奏 |
| 对白质量 | 对话自然度 |
| 情绪张力 | 情绪起伏 |
| 设定一致性 | 与知识库/设定是否吻合 |

**③ 设定一致性检查**

AI 会检查你的故事中是否有前后矛盾的地方：
- 角色名字是否一致（比如前面叫"张三"，后面变成"张山"）
- 地点描述是否一致
- 时间线是否合理

**④ 审查面板（嵌入区）**

| 审查类型 | 做什么 | 需要什么 |
|----------|--------|----------|
| 规则审查 | 内置规则引擎 | 不需要额外模型调用 |
| 语义审查 | LLM 深度分析 | Claude Code + 活跃模型凭据 |
| 规则+语义 | 两者结合 | 同上 |

审查报告可转为 **修改计划**，在「怎么改」组中跟进。

> **什么时候做质量检查？** 不必每章都查。建议写完一个阶段（如 10 章或一卷）做一次；日常用「今日建议」的轻量诊断即可。

## 11.2 悬念分析

### 什么是悬念分析？

悬念分析帮你检查故事的"吸引力"——每一章的悬念够不够强，故事的张力曲线是不是合理。

### 怎么打开？

点击右侧导航栏的 **"悬念分析"**

### 你会看到什么？

**① 每章的悬念强度**

每一章都会有一个悬念评分，用颜色标识：
- 🔴 红色：高悬念（很好！）
- 🟡 黄色：中等悬念（还行）
- 🟢 绿色：低悬念（可能太平淡了）

**② 张力曲线**

一个柱状图，展示全书的悬念走势。

**怎么看？**
- 理想状态是**波浪形**——有高有低，像心电图一样
- 如果一直很平，说明故事缺乏起伏
- 如果某一章突然掉下来了，说明那一章可能太平淡

**③ 悬念线索分类**

AI 会识别你故事中的悬念线索属于哪种类型：
- **悬疑类**：谁是凶手？真相是什么？
- **危险类**：主角会不会死？他们能逃出去吗？
- **关系类**：他们会不会在一起？友情会不会破裂？
- **揭露类**：这个秘密什么时候会被发现？

**④ 高潮时刻**

AI 会标记出每一章的最高悬念点——就是最让读者紧张、最想知道后面发生什么的地方。

> **怎么看悬念分析？** 如果你的张力曲线一直很平，试着在某些章节增加冲突或悬念。如果某一章的悬念评分很低，考虑在那一章末尾加一个"钩子"（一个让读者想继续看下去的悬念）。

# 第十二章：规划后续章节

> Story OS 中「写什么」组的两个入口：**今日建议**（每天干什么）与 **创作路线**（中长期章表）。使用前建议先完成同步。

## 12.1 今日建议与同步

### 快速同步 vs 全书理解

打开 **今日建议** 页，顶部有两个同步按钮：

| 按钮 | 作用 | 耗时 | 适用场景 |
|------|------|------|----------|
| **快速同步** | 扫描近期章节与知识库，刷新今日任务与诊断 | 较快 | 日常写作前 |
| **全书理解** | 深度分析全书结构与理解层 | 较慢 | 大改前、规划新卷前 |

同步完成后你会看到：

**① 今日任务** — 按优先级排列的写作任务，带预估时间，可点「开始写作」跳转工作台。

**② 诊断建议** — AI 分析已写内容的改进点；可生成 **行动计划** 或 **修改计划**。

若显示「暂无今日任务」，请先点 **快速同步**，或在 **创作路线** 中生成章节规划。

## 12.2 创作路线

### 什么是创作路线？

创作路线就是你的**章节路线图**——告诉你每一章应该写什么，目前写到哪了。

### 怎么打开？

点击右侧导航栏的 **"创作路线"**

### 你会看到什么？

**① 故事目标**

- **当前状态**：你的故事现在发展到哪了
- **目标状态**：故事最终要到哪
- **重大变化**：中间会发生什么关键转折
- **成功标准**：怎样算写完了

**② 章节路线图**

一个网格，每一章是一张卡片。卡片上显示：
- 章节号
- 标题提示
- 这一章的目的（推动什么剧情）
- 钩子（吸引读者的点）
- 状态标识：
  - 灰色 = 计划中（还没写）
  - 蓝色 = 进行中（正在写）
  - 绿色 = 已完成
  - 黄色 = 已跳过

**③ 生成创作路线**

点击 **"生成创作路线"** 按钮，AI 会根据你的大纲自动规划后续章节。

你可以选择规划范围（如 5 / 10 / 20 / 50 章）。生成后每章一张卡片，状态包括：计划中、进行中、已完成、已跳过。

## 12.3 节拍大纲

### 什么是节拍大纲？

节拍大纲帮你把一章拆成多个小场景（节拍），然后按节拍来写。

### 什么时候用？

当你觉得"写一整章太难了"的时候。先拆成小场景，一个一个写，压力小很多。

### 怎么用？

1. 点击右侧导航栏的 **"节拍大纲"**
2. 选择一个章节
3. 点击 **"添加节拍"**
4. 填写：
   - **标题**：这个场景叫什么
   - **场景类型**：动作 / 对话 / 蒙太奇 / 转场
   - **描述**：这个场景发生了什么
5. 重复步骤 3-4，把一章拆成多个场景

### 写作时使用

列好节拍后，点击 **"一键从节拍写作"** 按钮。AI 会根据你的节拍大纲生成写作计划，然后你可以按计划一个场景一个场景地写。

# 第十三章：编辑和审稿

## 13.1 修改计划

### 什么是修改计划？

当 AI 或你自己发现了问题，需要修改时，"修改计划"帮你管理这些改动。

### 修改计划从哪来？

- 来自"今日建议"中的诊断建议
- 来自"作品质量"中的审查结果
- 来自你手动创建的计划

### 怎么管理修改计划？

1. 点击右侧导航栏的 **"修改计划"**
2. 你会看到三个状态的计划：

| 状态 | 含义 |
|------|------|
| 待确认 | AI 提出了修改建议，等你确认要不要改 |
| 执行中 | 你已经开始改了 |
| 已完成 | 改完了 |

3. 点击一个计划查看详情
4. 确认后点击 **"在工作台执行"**，跳转到工作台开始修改
5. 改完后点击 **"标记完成"**

## 13.2 审稿中心

### 什么是审稿中心？

审稿中心是一个**跨项目的深度审查工具**。它比"作品质量"页面的审查更深入、更专业。

### 怎么打开？

点击左侧导航栏的 **"审稿中心"**（图标 ◑）

### 怎么使用？

1. 在顶部的**项目选择器**中选择你要审查的项目
2. 你会看到两个审查面板

**EngineCriticPanel（规则引擎审查）：**

有几种审查模式：

| 模式 | 做什么 |
|------|--------|
| 规则审查 | 用内置规则引擎检查（不需要 AI） |
| 语义审查 | 用 LLM 深度分析（需要 AI 模型） |
| 规则+语义 | 两者结合 |

点击一个模式，等待审查完成。

**审查结果包含：**
- **Governor 决策**：通过(APPROVE) / 需修订(REVISE) / 驳回(REJECT)
- **6 维度评分**：每个维度是 HARD（严重问题）/ SOFT（小问题）/ PASS（通过）
- **问题列表**：所有发现的问题，每个有具体位置和修改建议
- **修订任务**：可以转换为修改计划

> **审稿中心 vs 作品质量的区别：** 作品质量页面的审查更轻量，适合日常检查。审稿中心更深入，适合阶段性深度审查（比如写完一卷后）。

# 第十四章：编剧专用功能

> **注意：** 这一章的内容只适用于项目类型为"电影剧本"、"剧集剧本"或"短视频脚本"的项目。如果你的项目是小说，可以跳过这一章。

## 14.1 电影剧本

如果你的项目类型是"电影剧本"，进入工作台后，中央编辑区会变成一个**场景看板**。

### 场景看板是什么？

场景看板把你的剧本按**三幕结构**组织：

| 幕 | 名称 | 内容 |
|----|------|------|
| 第一幕 | 建置 | 故事的开端，介绍角色和世界观 |
| 第二幕 | 对抗 | 故事的主体，冲突和挑战 |
| 第三幕 | 解决 | 故事的结尾，高潮和结局 |

每个场景是一张卡片，显示：
- 内景/外景（INT/EXT）
- 地点
- 时间（白天/夜晚等）
- 出场角色
- 故事线
- 梗概
- 预估时长

### 怎么操作？

**添加场景：**
1. 点击 **"添加场景"** 按钮
2. 填写场景信息（内/外景、地点、时间、幕次、梗概、时长）
3. 保存

**编辑场景：**
- 点击场景卡片，打开详细编辑面板

**排序场景：**
- 直接**拖拽**场景卡片，改变顺序

**筛选场景：**
- 使用"故事线图例"按颜色筛选不同故事线的场景

### 角色统计

切换到 **"角色统计"** 标签页，可以看到每个角色的出场时间等统计。

### 导出为 Fountain 格式

电影剧本可以导出为 **Fountain 格式**——这是剧本行业的标准交换格式，可以在 Final Draft 等专业软件中打开。

## 14.2 剧集剧本

如果你的项目类型是"剧集剧本"，你会看到三个标签页：

**① 剧集列表**
- 管理每一集
- 每集包含多个场景
- 支持创建/编辑/删除/排序

**② 伏笔追踪**
- 管理跨集的叙事伏笔
- 追踪伏笔在哪一集埋设、在哪一集回收

**③ 角色弧线**
- 追踪角色在整部剧中的成长轨迹
- 每个角色一条弧线，标注在哪些集有关键发展

## 14.3 短视频脚本

如果你的项目类型是"短视频脚本"，你会看到一个**分镜看板**。

**功能：**
- 管理每个镜头
- 支持分段和平台设置（抖音、B站等）
- 节奏分析：帮你控制视频的节奏

## 14.4 编剧室

### 什么是编剧室？

编剧室是一个专业编剧工具，帮你管理世界观规则和自动化修订流程。

### 怎么打开？

点击右侧导航栏的 **"编剧室"**

### 编剧室能做什么？

**① Canon 规则**

Canon 规则是你故事世界的"铁律"。比如：
- "这个世界的魔法只能用来治愈，不能用来攻击"
- "主角永远不能杀人"
- "时间旅行只能回到过去，不能去未来"

设定这些规则后，AI 在帮你写作时会自动遵守。如果 AI 写了违反规则的内容，系统会自动提醒。

**② 结构化记忆**

帮你管理每一章的叙事摘要和结构化事实。这样 AI 在写后面的章节时，能准确记住前面发生了什么。

**③ 流水线（Creator → Critic → Governor）**

编剧室可启动自动修订流水线：

1. 在 **编剧室** 或 **今日建议** 中触发 Critic / 自动流水线
2. **Critic** 产出问题与维度评分
3. **Governor** 给出决策：通过 / 建议修订 / 建议重写
4. 若需修订，系统自动在 **修改计划** 中生成任务（来源标记为「编剧室」）
5. 到 **修改计划** 确认 → **在工作台执行** → 完成后标记完成

**④ 流水线记录**

编剧室会保留每轮修订历史，便于对比 Governor 备忘录与后续改稿。

# 第十五章：导出和备份你的作品

## 15.1 导出你的作品

当你想把作品从文匠 Studio 里拿出来（比如投稿、分享给别人、做备份），就需要用到导出功能。

### 怎么导出？

**方法一：** 在工作台中，点击左侧菜单的 **"导出"** 图标

**方法二：** 在工作台顶部工具栏中，点击**导出按钮**

会弹出一个导出模态框。

### 导出格式选择

**项目级导出（整个项目）：**

| 格式 | 包含什么 | 适合什么场景 |
|------|----------|-------------|
| ZIP 包 | 所有 Markdown 文件、大纲、设定 | 备份整个项目（推荐） |
| EPUB | 电子书格式 | 在手机/平板上阅读 |
| Word 全书 | 所有章节合并成一个 Word 文档 | 投稿 |
| ZIP 含分章 Word | 每章一个 Word 文件的压缩包 | 逐章投稿 |

**章节级导出（单个章节）：**

| 格式 | 适合什么场景 |
|------|-------------|
| Markdown (.md) | 纯文本，最通用 |
| Word (.docx) | 投稿 |

> **关于 Word 导出：** Word 导出需要安装 Python 依赖。如果按钮是灰色的不可点击，说明依赖没装好。模态框里会显示安装说明。

### 怎么操作？

1. 选择导出范围（项目/章节）
2. 选择导出格式
3. 点击 **"导出"** 按钮
4. 等待处理完成
5. 浏览器会自动下载导出的文件

## 15.2 导入文档

如果你有现成的文档想导入到项目中：

### 怎么导入？

1. 在工作台中，点击顶部工具栏的 **"上传"** 按钮
2. 会弹出一个导入模态框
3. 选择你要导入到哪个目录：
   - **正文**：手稿章节
   - **大纲**：大纲文件
   - **设定集**：设定文件
   - **旧稿归档**：旧版本
   - **试验稿**：试验性修改
4. 点击 **"选择文件"** 或直接把文件**拖拽**到模态框中
5. 等待上传和转换完成

### 支持的格式

| 格式 | 说明 |
|------|------|
| .md | Markdown 文件，直接导入 |
| .txt | 文本文件，自动转换为 Markdown |
| .docx | Word 文件，自动转换为 Markdown |
| .pdf | PDF 文件，自动转换为 Markdown |
| .html | 网页文件，自动转换为 Markdown |
| .zip | 压缩包，自动解压并导入 |

> **导入 Word 文件：** Word 导入需要 Python 依赖。如果转换失败，检查一下 Python 环境是否正确配置。

## 15.3 版本管理（备份和恢复）

### 什么是版本管理？

版本管理就是给你的项目"拍照存档"。在做重大修改前，先拍一张"快照"。如果改坏了，可以恢复到之前的状态。

### 怎么创建版本快照？

1. 点击左侧导航栏的 **"项目版本"**（图标 ⧉）
2. 选择你的项目
3. 点击 **"创建快照"** 按钮
4. 填写：
   - **标题**：比如"改写前备份"、"第一卷完成"
   - **备注**：可选，写一下这个版本的情况
5. 点击保存

### 怎么恢复版本？

1. 在版本列表中找到你想恢复的版本
2. 点击 **"查看 Diff"**，看看这个版本和现在的区别
3. 确认后点击 **"恢复到工作台"**

### 什么时候该存版本？

- 删除重要角色之前
- 大改剧情线之前
- 完成一个阶段之后（比如写完第一卷）
- 准备做实验性修改之前

## 16.1 全局搜索

### 什么是全局搜索？

全局搜索可以搜索你整个项目的所有文件——正文、大纲、设定、归档，全部都能搜。

### 什么是素材中心？

素材中心是一个**跨项目的素材库**。如果你同时写多部作品，可以把通用素材放在这里。

### 怎么用？

1. 点击左侧导航栏的 **"素材中心"**（图标 ◧）
2. 选择一个项目
3. 你会看到一个素材列表
4. 可以添加/编辑/删除素材

### 素材类型

| 类型 | 适合放什么 |
|------|-----------|

## 16.3 本机技能与发现安装

除模型凭据外，叙事引擎的行为由 **Skill** 决定：

1. AI 中心 → **本机技能**：设置默认 Skill、literary-writer 根路径
2. AI 中心 → **发现安装**：浏览社区或本地 Skill 并安装
3. 保存后新对话即按新 Skill 工作

详见 [3.4 配置本机技能](#34-配置本机技能叙事引擎)。

## 16.4 项目分享

多人协作时，可将项目共享给其他注册用户。

### 怎么分享？

1. 在项目库中，点击项目卡片右上角 **⋯**
2. 选择 **「共享权限」**（需项目管理权限）
3. 勾选用户并设置 **只读** 或 **可编辑**
4. 保存

## 16.5 MCP 扩展

### 什么是 MCP？

MCP（Model Context Protocol）是一种让 AI 连接外部工具的协议。通过 MCP 扩展，你可以给 AI 增加新能力。

### 怎么管理？

1. 点击左侧导航栏的 **"AI 中心"**
2. 点击 **"MCP 扩展"** 标签页
3. 你可以：
   - 查看已安装的 MCP 服务器
   - 启用/禁用服务器
   - 搜索和安装新的 MCP 服务器
   - 测试服务器连接

## 16.6 用户管理（管理员功能）

如果你是管理员，可以管理其他用户：

1. 点击左侧导航栏的 **"用户管理"**（图标 👤）
2. 你可以：
   - 创建新用户
   - 启用/禁用用户
   - 删除用户

## 16.7 留言板

留言板是一个公共互动空间，可以和其他用户交流创作心得。

1. 点击左侧导航栏的 **"留言板"**（图标 💬）
2. 你可以：
   - 发布留言（支持图片，最多 4 张）
   - 回复别人的留言
   - 匿名留言

2. 本机安装 [Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code)，确保终端能执行 `claude -p "hello"`
3. **Docker 部署**：确认镜像内已包含 Claude Code，且容器有权执行
4. 安装或更新后 **重启文匠 Studio 服务**（或重建容器）
5. 总览显示「已连接」后，再回到工作台重连对话

## Q：模型连接测试失败怎么办？

按顺序排查：

| 步骤 | 检查项 |
|------|--------|
| 1 | **协议与 Base URL 是否匹配**：`/v1` 结尾用 OpenAI 兼容；`/anthropic` 用 Anthropic 兼容 |
| 2 | **API Key** 是否正确、有余额、未过期 |
| 3 | 点击 **测试连接** 看具体错误（404 多为 URL/协议错；401 多为 Key 错） |
| 4 | Docker 环境：容器能否访问外网 API |
| 5 | MiMo 用户：优先试 **MiMo Anthropic** 模板，Base URL 应为 `…/anthropic` |

配置成功后务必 **设为活跃**，并 **新建对话会话** 验证。

## Q：AI 不回复我怎么办？

**检查步骤：**
1. 看对话面板顶部：须为 **Claude Code 已连接**（绿色）
2. 若红色「连接失败」，点 **重连**
3. 去 AI 中心 → 总览：Claude Code + 活跃模型是否都正常
4. 检查 API Key、网络、容器出网
5. 查看浏览器控制台或服务端日志是否有 WebSocket 断开

## Q：换了 MiMo / DeepSeek 模型，AI 还说自己是 Claude？

旧会话可能绑定了之前的模型上下文。

1. 确认新模型已 **设为活跃** 且 **测试连接成功**
2. 在会话选择器点 **「+ 新建会话」**
3. 在新会话问：「你是什么模型？由谁提供？」
4. 若仍不对，完全 **清空当前会话** 或退出项目重进

## Q：我的内容丢失了怎么办？

**文匠 Studio 有自动保存功能**，停止打字 2.5 秒后会自动保存。如果你的内容丢失了：
1. 检查左侧文件列表，看看文件是否还在
2. 去"项目版本"页面，看看是否有之前的快照可以恢复
3. 如果都没找到，检查 `data/projects/你的项目ID/workspace/正文/` 目录下是否有 `.md` 文件

## Q：AI 写的内容有"AI 味"怎么办？

这是正常的。AI 生成的内容有时候会有一些特征性的表达（比如过于工整、缺乏个性）。

**解决方法：**
1. 不要直接使用 AI 的输出，把它当初稿
2. 用"润色"功能让 AI 帮你优化
3. 自己手动修改，加入你的个人风格
4. 在角色工坊中训练"声纹DNA"，让 AI 的对话更自然

## Q：怎么同时管理多个项目？

1. 在项目库中创建多个项目
2. 每个项目是完全独立的
3. 在"创作看板"首页可以看到所有项目的统计数据
4. 在"素材中心"可以管理跨项目的共享素材

## Q：导出 Word 失败怎么办？

Word 导出需要 Python 依赖。如果失败：
1. 检查是否安装了 Python 3.9+
2. 检查是否安装了必要的 Python 包
3. 导出模态框里会显示具体的错误信息和安装说明

## Q：怎么修改项目信息（标题、类型等）？

1. 进入项目工作台
2. 点击顶部工具栏的**设置按钮**（⚙️ 图标）
3. 在弹出的模态框中修改信息
4. 保存

## Q：怎么删除项目？

1. 在项目库中，点击项目卡片 **⋯** 菜单
2. 选择 **「删除」**
3. 输入项目标题确认（防误删）
4. 确认删除

> **注意：** 删除不可恢复！建议先导出 ZIP 备份。

## Q：Docker 部署后数据存在哪里？

数据保存在 Docker **数据卷**（如 `literarycraft-data`），挂载到容器内项目数据目录。重建容器通常不丢数据；**删除数据卷会清空所有项目**。升级前建议先打 **项目版本** 快照或导出 ZIP。

## Q：今日建议显示「暂无任务」？

1. 先点 **快速同步**
2. 确认项目已有章节或知识库内容
3. 到 **创作路线** 生成章节规划
4. 再回到今日建议刷新

## Q：键盘快捷键有哪些？

| 快捷键 | 功能 |
|--------|------|
| Cmd/Ctrl + S | 保存当前章节 |
| Cmd/Ctrl + ← | 切换到上一章 |
| Cmd/Ctrl + → | 切换到下一章 |
| Cmd/Ctrl + K | 全局搜索 |
| F | 专注模式（隐藏侧栏） |
| ? | 查看所有快捷键 |
| Enter | 发送 AI 消息 |
| Shift + Enter | 在 AI 输入框中换行 |
| Escape | 关闭弹窗 / 退出专注模式 |
| 选中 ≥8 字 | 浮动操作栏：改写 / 扩写 / 润色 / 与 AI 讨论 |

## Q：文匠 Studio 支持多人协作吗？

支持。在项目库通过 **⋯ → 共享权限** 邀请其他用户，设置只读或可编辑。管理员还可在 **用户管理** 中创建账号。

## Q：我的数据存在哪里？

| 部署方式 | 数据位置 |
|----------|----------|
| 本机开发/运行 | 项目目录下 `backend-node/data/`（项目、用户、对话、模型配置等） |
| Docker | 命名数据卷挂载进容器，具体卷名见 `docker-compose.yml` |

数据由你控制的服务端保存，**不会上传到文匠 Studio 官方云端**。团队部署时请自行备份数据卷。

## Q：本机技能和默认 Skill 有什么用？

Skill 定义叙事引擎的创作流程（如网文规划、章节审查脚本）。在 AI 中心 → **本机技能** 设置默认项后，所有新对话按该 Skill 执行。更换 Skill 后同样建议新建会话。
