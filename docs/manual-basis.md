# 文匠 Studio — 功能逻辑与业务流程全景文档

> 本文档是编写使用手册的基础材料，覆盖产品全部功能模块、用户操作流程和业务逻辑。

---

## 一、产品概述

**文匠 Studio** 是一款**本地优先、AI 驱动**的长篇叙事创作工作台，专为网文作者、编剧和长篇小说家设计。

### 核心能力

| 能力 | 说明 |
|------|------|
| AI 续写/改写/润色 | 基于 WebSocket 流式生成，支持选中文本内联编辑 |
| 故事知识库 | 自动提取角色、关系、时间线、伏笔、地点，构建结构化知识图谱 |
| RAG 语义检索 | 基于 LanceDB 向量搜索，AI 对话和写作时自动召回最相关章节片段 |
| 编剧引擎 | 支持电影剧本（三幕结构）、剧集剧本（分集分场）、短视频分镜 |
| 质量审查 | 8 维度审查（设定一致性、时间线、叙事连贯性、角色一致性、逻辑、AI味、改写检查、角色行为约束） |
| 版本管理 | 章节级快照、Diff 对比、一键回滚 |
| 多格式导出 | ZIP/DOCX/EPUB/Fountain |
| 多模型支持 | OpenAI、Anthropic Claude、DeepSeek、Gemini 及任意 OpenAI 兼容 API |

### 支持的作品类型

| 类型代码 | 说明 |
|----------|------|
| `novel_long` | 长篇小说 |
| `novel_short` | 短篇小说 |
| `screenplay_film` | 电影剧本 |
| `screenplay_series` | 剧集剧本 |
| `web_short` | 短视频脚本 |

### 创作模式

| 模式 | 说明 |
|------|------|
| `scratch` | 从零创作 |
| `rewrite` | 改写/续写已有作品（支持旧稿归档、试验稿） |
| `import` | 导入已有文档开始创作 |

## 二、系统架构（用户视角）

### 2.1 应用层级

```
┌─────────────────────────────────────────────────────┐
│                    文匠 Studio                        │
├──────────┬──────────────────────────────────────────┤
│  全局层   │  创作看板 · 项目库 · 审稿中心 · 素材中心   │
```

### 2.2 导航结构

#### 全局左侧导航栏

| 图标 | 名称 | 路由 | 说明 |
|------|------|------|------|
| ◉ | 创作看板 | `/` | 全局仪表盘，写作统计总览 |
| ◫ | 项目库 | `/projects` | 项目列表，创建/管理/搜索项目 |
| ◑ | 审稿中心 | `/review` | 跨项目审稿，规则引擎 + LLM 审查 |
| ◧ | 素材中心 | `/assets` | 跨项目素材管理（角色卡、地点、灵感） |

| 💬 | 留言板 | `/guestbook` | 公共留言互动 |
| 👤 | 用户管理 | `/users` | 仅管理员可见，用户增删改查 |

#### 项目内右侧导航栏（Story OS）

进入项目工作台后，右侧出现**故事工程导航**，分三组：

**"写什么"**
- 今日建议 — AI 生成的每日写作任务
- 创作路线 — 章节路线图与里程碑

**"写得好吗"**
- 作品质量 — 健康评分与一致性检查
- 悬念分析 — 每章悬念强度可视化
- 节拍大纲 — 场景节拍卡编辑
- 作品知识 — 角色/关系/时间线/伏笔/地点

**"怎么改"**
- 修改计划 — 待确认/执行中/已完成的修改计划
- 角色工坊 — 角色心理模型、声纹DNA、关系图
- 设定圣经 — 结构化世界观文档
- 编剧室 — Canon 规则、结构化记忆、流水线历史

## 三、用户账户与认证

### 3.1 注册

- 访问 `/login` 页面，切换到"注册账号"标签
- 填写：用户名、显示名称、密码、确认密码
- 注册成功后自动登录，跳转到创作看板
- 管理员可在后台控制是否开放注册

### 3.2 登录

- 输入用户名和密码
- 登录成功后存储 JWT Token 到 localStorage
- 支持跳转回登录前的目标页面

### 3.3 用户角色

| 角色 | 权限 |
|------|------|
| 普通用户 | 创建/编辑自己的项目，使用所有创作功能 |
| 管理员 | 以上 + 用户管理 + 技能安装 + 技能目录更新 |
| 超级管理员 | 以上 + 留言板删除 + 不可被禁用/删除 |

### 3.4 退出登录

- 点击右上角头像 → 下拉菜单 → "退出登录"

## 四、创作看板（仪表盘）

**路由：** `/`

### 4.1 功能说明

创作看板是登录后的默认首页，提供全局写作活动概览。

### 4.2 界面元素

**指标卡片区（5 张卡片）：**
- 项目总数
- 今日活跃项目数
- 今日更新章节数
- 今日字数变化
- 累计总字数

**7 日趋势图：**
- 柱状图展示每日写作活动
- 每根柱子高度 = 当日更新字数
- 柱子标签 = 当日更新章节数
- 鼠标悬停显示详细统计

**停滞项目列表：**
- 显示超过 14 天未更新的项目
- 点击可跳转到该项目的健康页面

**按项目统计表：**
- 列：项目名称、今日更新章节、今日字数变化、累计章节、累计字数
- 活跃行高亮显示

## 五、项目库

**路由：** `/projects`

### 5.1 功能说明

项目库是所有创作项目的管理中心，支持创建、搜索、筛选、排序、批量操作。

### 5.2 界面布局

**顶部横幅：**
- 品牌介绍
- Claude Code 连接状态徽章

**工具栏：**
- 视图切换：网格/列表
- 排序选择器：多种排序方式
- 搜索栏：按标题搜索
- 筛选标签：按作品类型、创作模式、题材、状态筛选
- 批量模式开关
- "新建项目"按钮

**继续创作栏：**
- 显示最近打开的项目，一键跳转

### 5.3 创建项目

点击"新建项目"打开创建模态框：

| 字段 | 说明 |
|------|------|
| 标题 | 项目名称（必填） |
| 作品类型 | 长篇小说/短篇小说/电影剧本/剧集剧本/短视频 |
| 创作模式 | 从零创作/改写续写/导入已有 |
| 改写方向 | 改写模式下的改写说明（可选） |
| 题材/风格 | 如玄幻、都市、科幻等（可选） |
| 简介 | 项目简介（可选） |

### 5.4 项目卡片操作

每个项目卡片/列表项提供右键菜单：

| 操作 | 说明 |
|------|------|
| 置顶/取消置顶 | 固定项目到列表顶部 |
| 重命名 | 修改项目标题 |
| 编辑简介 | 修改项目简介 |
| 设置状态 | 写作中/草稿/修订中/已完成 |
| 归档/取消归档 | 归档后隐藏，可恢复 |
| 删除 | 需输入标题确认，不可恢复 |
| 导入 | 上传文档到项目 |
| 分享 | 管理项目共享权限 |

### 5.5 批量操作

- 开启批量模式后可选择多个项目
- 支持批量删除

### 5.6 项目分享

- 项目所有者可邀请其他用户
- 权限级别：只读 / 可编辑
- 通过复选框和角色下拉框管理

写作工作台是核心创作界面，采用三栏布局，集成编辑器、资源管理和 AI 对话。

### 6.2 界面布局

```

│ 文稿  │  [工作区头部工具栏]      │  [会话选择器]      │
│ 大纲  │  [创作进度卡]           │  [连接状态栏]      │
│ 设定  │  [Markdown 编辑器]     │  [消息历史]        │
│ 试验  │                      │  [快捷提示]        │
│ 旧稿  │                      │  [消息输入框]       │
│ 导出  │                      │                  │
└──────┴──────────────────────┴──────────────────┘
```

**移动端：** 折叠为标签页界面（编辑/资源/对话）

### 6.3 左侧菜单

| 图标 | 名称 | 说明 |
|------|------|------|
| 📄 | 文稿 | 正文章节文件列表 |
| 📋 | 大纲 | 大纲文件列表 |
| ⚙️ | 设定 | 设定集文件列表 |
| 🧪 | 试验 | 试验稿文件列表（改写模式可见） |

### 6.4 资源侧栏

- 根据左菜单选中项显示对应文件列表
- 点击文件加载到中央编辑器
- 可折叠/展开

### 6.5 工作区头部

| 元素 | 说明 |
|------|------|
| 返回链接 | 返回项目库 |
| 项目标题 | 当前项目名称 |
| 上下文选择器 | 选择注入 AI 的上下文范围 |
| 设置按钮 | 打开项目设置模态框 |
| 上传按钮 | 打开导入模态框 |
| 快捷键按钮 | 打开快捷键参考 |
| 诊断按钮 | 打开项目诊断报告 |
| 导出按钮 | 打开导出模态框 |

### 6.6 编辑器（ReaderPane）

**两种模式：**
- **编辑模式：** 纯文本 `<textarea>` 编辑
- **预览模式：** 简单 Markdown 渲染

**功能特性：**
- 四种字号可选（15px-20px），持久化到 localStorage
- 自动保存：停止编辑 2.5 秒后自动保存
- 脏状态守卫：离开页面前提醒保存
- 章节导航：Cmd/Ctrl + 左右箭头切换章节
- 大纲模式：当内容来源为"大纲"时，编辑器上方显示大纲树面板

**选中文本操作栏：**

选中至少 8 个字符后，出现浮动操作栏：

| 操作 | 说明 |
|------|------|
| AI 改写 | 对选中文本进行 AI 改写 |
| 扩写 | 对选中文本进行扩写 |
| 润色 | 对选中文本进行润色 |
| 与 AI 讨论 | 将选中文本作为引用发送到对话面板 |
| 取消 | 关闭操作栏 |

### 6.7 AI 对话面板（ChatPanel）

**连接状态：**
- 显示 WebSocket 连接状态（已连接/连接中/错误）
- 连接失败时显示重连按钮

**会话管理：**
- `SessionSelector` 下拉框：创建/切换/删除会话
- 会话按章节绑定或项目全局
- 切换章节时自动聚焦到对应章节会话

**上下文详情抽屉：**
- 显示当前注入的项目上下文：章节数、活跃技能、会话记忆等

**快捷提示按钮：**
- 根据项目类型显示不同的快捷提示（如"续写"、"分析角色"等）

**消息输入框：**
- Enter 发送，Shift+Enter 换行
- 支持流式响应，实时显示 AI 生成内容

**消息类型：**
- 普通对话
- 写作计划执行卡
- 写作流水线触发

### 6.8 编剧编辑器（ScreenplayEditor）

当项目类型为 `screenplay_film`、`screenplay_series` 或 `web_short` 时，编辑器替换为专业编剧界面。

#### 电影剧本（screenplay_film）

**两个标签页：**

**场景看板（Scene Board）：**
- 三列看板：第一幕（建置）、第二幕（对抗）、第三幕（解决）
- 场景卡片显示：内景/外景、地点、时间、角色、故事线、梗概、时长估算
- 场景可拖拽排序
- 故事线图例：按颜色筛选场景
- 添加场景表单：内/外景、地点、时间、幕次、梗概、时长
- 点击场景打开详细编辑面板

**角色统计（Character Stats）：**
- 每个角色的出场时间等统计指标

#### 剧集剧本（screenplay_series）

**三个标签页：**

**剧集列表（Episodes）：**
- 管理剧集，每集包含多个场景
- 支持创建/编辑/删除/排序剧集和场景

**伏笔追踪（Foreshadows）：**
- 管理跨集叙事伏笔
- 支持创建/编辑/更新伏笔

**角色弧线（Character Arcs）：**
- 追踪角色跨集发展

#### 短视频（web_short）

**分镜看板（Shot Board）：**
- 管理单个分镜
- 支持分段和平台设置
- 节奏分析功能

### 6.9 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| Cmd/Ctrl + S | 保存当前章节 |
| Cmd/Ctrl + ←/→ | 切换上/下一章 |
| Cmd/Ctrl + K | 全局搜索 |

## 七、Story OS — 故事工程系统

Story OS 是项目内的结构化叙事工程系统，通过右侧导航栏访问。所有 Story OS 页面共享统一的子导航（StorySubnav），包含"返回工作台"链接和三个场景分组标签。

### 7.1 今日建议

**路由：** `/projects/:projectId/suggestions`

**功能：** 每日创作指挥中心，AI 生成的写作任务和诊断建议。

**界面：**
- 顶部：项目标题、路线图摘要、创作目标（当前状态→目标状态）
- 视野选择器：5/10/20/50 章
- 操作按钮："快速同步"、"深度理解"、"一键章节流水线"

**两个标签页：**

**今日任务：**
- 按优先级排序的写作任务列表
- 每个任务显示预估时间
- "开始写作"按钮跳转到工作台并携带写作计划

**诊断建议：**
- 手稿中发现的问题（弧线增强、冲突提升、伏笔回收、章节规划）
- 每个建议有"开始写作"按钮，生成行动计划并跳转工作台

**写作流水线向导（WritePipelineWizard）：**

三步引导流程：
1. 选择今日任务
2. 进入工作台写作
3. 运行审查

编剧项目额外支持"编剧室流水线"选项（自动修订，Governor 审批，最多 2 轮）。

### 7.2 创作路线

**路由：** `/projects/:projectId/roadmap`

**功能：** 章节路线图与里程碑管理。

**界面：**
- 故事目标：当前状态、目标状态、重大变化、成功标准
- 章节路线图网格：每张卡片显示章节号、标题提示、目的、钩子、状态
- 状态标识：计划中/进行中/已完成/已跳过
- 视野选择器
- "生成创作路线"按钮

### 7.3 作品质量

**路由：** `/projects/:projectId/health`

**功能：** 质量诊断与健康评分。

**包含组件：**
- `EngineCriticPanel`：规则引擎 + Governor 决策系统
- `ReviewPanel`：启发式审查面板

**功能详情：**
- 健康评分（6 维度）
- 设定一致性检查（角色/地点名称验证）
- 创作验证（对最新章节运行验证，自动同步结构化记忆）
- 验证统计（通过率、最近记录含检查详情）
- 总体质量评分
- 按章节评分含维度分解

**审查维度（6 维度）：**

| 维度 | 说明 |
|------|------|
| 结构 | 故事结构完整性 |
| 角色 | 角色塑造与一致性 |
| 场景 | 场景描写与转换 |
| 叙事压力 | 悬念、张力、节奏 |
| 声音/对话 | 角色语言风格 |
| 连贯性/主题 | 主题表达与前后连贯 |

**Governor 决策：**
- APPROVE（通过）/ REVISE（修订）/ REJECT（驳回）
- 附带修订备忘录

### 7.4 悬念分析

**路由：** `/projects/:projectId/suspense`

**功能：** 每章悬念强度可视化分析。

**界面元素：**
- 悬念强度评分和等级
- 叙事压力（总体/状态/方向）
- 悬崖检测
- 开放/已解决线索计数
- 张力曲线可视化（柱状图，颜色编码强度）
- 高潮时刻引文
- 悬念线索分类：悬疑/危险/关系/揭露
- 主题检测含强度评分
- 伏笔扫描（铺垫/回收标记）

### 7.5 节拍大纲

**路由：** `/projects/:projectId/beats`

**功能：** 结构化场景节拍卡编辑器。

**操作流程：**
1. 选择单元（章节）编号
2. 设置标题和描述
3. 添加/删除/排序节拍
4. 每个节拍：标题、场景类型（动作/对话/蒙太奇/转场）、描述
5. "一键从节拍写作"按钮生成写作计划并跳转工作台

**预览功能：**
- 可折叠的"预览创作者上下文链"
- 显示将发送给 AI 的完整提示词

### 7.6 作品知识

**路由：** `/projects/:projectId/knowledge`

**功能：** 结构化故事理解中心。

**三个主要区域 + 查询界面：**

**角色卡片（Character Cards）：**
- 可编辑角色卡片网格

## 十六、AI 对话与写作流程

### 16.1 对话流程

```
用户输入消息
    ↓
WebSocket 发送 { type: "chat", projectId, message, sessionId }
    ↓
后端编排器（Orchestrator）处理：
    ├── 1. 构建项目上下文（标题、题材、章节数、大纲摘要、设定摘要、最新章节）
    ├── 2. 主编代理（Chief Editor）意图分类
    │      ├── 普通对话 → 继续
    │      ├── 写作计划 → 返回 story_plan 事件
    │      ├── 今日任务 → 返回 story_tasks_today 事件
    │      ├── 计划结果 → 返回 planner_result 事件
    │      └── 任务执行 → 返回 task_execute 事件
    ├── 3. 构建系统提示词（含项目专属指令）
    ├── 4. 组装上下文块
    │      ├── 故事知识库（Story KB）
    │      ├── RAG 检索结果
    │      ├── 用户选择的上下文（大纲、设定、知识、近期章节）
    │      ├── 焦点章节块
    │      ├── 记忆块
    │      └── 历史消息
    ├── 5. 调用 AI 模型（Claude CLI 或 HTTP）
    └── 6. 流式返回响应
    ↓
前端实时显示 AI 生成内容
```

### 16.2 写作流程

```
用户触发写作（快捷提示 / 写作流水线）
    ↓
WebSocket 发送 { type: "write", projectId, chapter, title?, outline? }
    ↓
后端处理：
    ├── 1. 上下文代理（context-agent）生成写作任务书
    │      → 5 段式：开篇委托、故事梗概、角色、写法指导、结尾位置
    ├── 2. 基于任务书起草章节正文
    ├── 3. 审查代理（reviewer）审核
    │      → 8 维度检查
    │      → 如有阻断问题(blocking=true)，必须修复才能继续
    ├── 4. 润色
    │      → 加载润色指南、排版、风格适配器
    │      → 修复非阻断问题
    │      → 反 AI 味最终检查
    ├── 5. 数据代理（data-agent）提取事实
    │      → 生成 3 个 JSON 工件
    │      → chapter-commit 持久化提交
    │      → 运行 5 个投影（状态/索引/摘要/记忆/向量）
    └── 6. Git 备份
    ↓
前端显示 WritePreviewModal（Diff 预览）
    → 用户选择：打开为新文件 / 替换当前内容
```

### 16.3 内联编辑流程

```
用户选中文本 → 浮动操作栏 → 选择操作（改写/扩写/润色）
    ↓
WebSocket 发送 { type: "inline_edit", projectId, selectedText, action, contextBefore, contextAfter }
    ↓
后端处理：发送选中文本 + 上下文到 AI
    ↓
前端显示 WritePreviewModal（替换预览）
    → 用户接受或拒绝
```

### 16.4 会话管理

- 会话按章节绑定（`bound_filename`）或项目全局（`__project__`）
- 消息溢出时自动归档，保留滚动窗口
- 每个会话追踪：记忆摘要、Claude 会话 ID、推理模型 ID
- 记忆摘要按消息数阈值启发式刷新

```
        正文/             ← 手稿章节 (.md)

            index.json    ← 会话索引含活跃会话 ID
```

## 二十三、完整用户旅程

### 23.1 新用户旅程

```
1. 访问 /login → 注册账号
2. 跳转到 / (创作看板) → 显示 OnboardingGuide（5步功能地图）
3. 导航到 /projects → 点击"新建项目"
4. 填写项目信息（标题、类型、模式、题材）→ 创建
5. 进入工作台 /projects/:id → 显示 WorkspaceOnboarding（空状态引导）
6. 导入 DOCX 或创建新章节
7. 使用右侧 ChatPanel 与 AI 对话，获取建议，触发写作
8. 通过 ProjectRightNav 访问 Story OS 功能
```

### 23.2 日常创作旅程

```
1. /login → 登录
2. / (创作看板) → 查看写作统计
3. /projects → "继续创作"链接跳转到最近项目
4. 工作台内：
   ├── 中央编辑器：编辑文本
   ├── 右侧 ChatPanel：与 AI 对话
   ├── 左侧资源栏：切换章节/大纲/设定
   └── 右侧导航：访问 Story OS 功能
5. 自动保存（2.5秒防抖）
6. 快捷键操作：Cmd+S 保存、Cmd+←/→ 切换章节、F 专注模式
```

### 23.3 管理员旅程

```
1. 同上 +
2. /users → 用户管理（创建/启用/禁用/删除用户）
3. 项目分享 → 管理项目共享权限
4. /ai/skills/discover → 安装新技能
```
