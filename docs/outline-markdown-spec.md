# 大纲 Markdown 规范（唯一真相源）

> 可视化大纲编辑器、Story OS Roadmap/Tasks 均 **只读写** `大纲/` 目录下的 Markdown，不维护平行数据库。

## 文件组织

- 推荐单文件：`大纲/总纲.md`
- 长篇可拆分：`大纲/第一卷.md`、`大纲/第二卷.md`
- 文件名任意，须为 `.md`

## 结构层级

| Markdown | 含义 |
|----------|------|
| `# 第一卷 · 卷名` | 卷（可选） |
| `## 第003章 章节标题` | 章（推荐四位序号） |
| `### 场景/节拍` | 章内小节（可选） |

章标题推荐格式：`第{序号}章 {标题}`，序号用于排序与 Task 关联。

## 梗概块

每章标题下第一段为 **梗概**（plain paragraph）。可视化编辑器「卡片视图」展示此段。

```markdown
## 第003章 夜访

林凡夜入苏府，首次见到苏清月。章末：发现玉佩与母亲遗物相同。
```

可选显式标记（解析器两种都支持）：

```markdown
## 第003章 夜访

> 梗概：林凡夜入苏府……
```

## 自由说明

- 卷/章标题行必须符合层级规范
- 标题下的 bullet、表格、引用块可作为写作备忘，解析时保留但不参与树节点排序

## 拖拽调序

编辑器调序后 **整段 Markdown 回写** 同一文件，保持 Git/版本 Diff 友好。

## 与正文关联

- 正文章节文件名：`正文/第0003章-夜访.md`（与大纲序号对齐为佳，非强制）
- Story Tasks 可读大纲下一未写章节作为建议
