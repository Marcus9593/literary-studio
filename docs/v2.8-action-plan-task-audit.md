## 炸点 4：Action 与 Plan 撞车

### 已观察模式

```text
Action.toPlan() 存在 → 命名上像 Action 承担 Plan 职责
```

**定性：** `createPlanFromAction` 是 **Action→Plan 工厂**，符合公约 2，但命名易误导 V3 开发。

### 禁止演化（V3 Director 前写入公约）

```text
❌ Action.execute()
❌ Action 内嵌 steps[] / execution_prompt 并直接 Execute（须落 Plan）
❌ Plan 完成后再自动生成 Action 再自动生成 Plan（循环）
❌ Task 直接从 Action 创建并跳过 Plan 确认（除显式 one-click 且审计）
```

### 允许链

```text
Understanding → Actions
Verify fail → Action (source: verify)
User → Action → Plan → Execute → Verify
Task → Plan → Execute
```
